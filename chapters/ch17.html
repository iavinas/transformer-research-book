Chapter 17

Vision Transformer (ViT)

For decades, convolutional neural networks owned computer vision. Every layer encoded a strong prior: nearby pixels matter more than distant ones, and the same pattern can appear anywhere in the image. Then in 2020, a team at Google asked a radical question — what if we threw away all of those priors and just fed image patches into a standard transformer? The answer was the Vision Transformer, and it changed the trajectory of the entire field. By the end of this chapter you will understand how ViT converts images into token sequences, why it needs massive data to outperform CNNs, how self-supervised methods like DINO and MAE eliminated that data dependency, and how the lineage from DINOv2 to DINOv3 produced universal visual features that rival human-designed representations.

The ViT Paper

The core insight of Dosovitskiy et al. (2020) is almost comically simple: split an image into fixed-size patches, flatten each patch into a vector, project it linearly into the transformer's embedding dimension, and process the resulting sequence with a standard transformer encoder — the same architecture used for language. No convolutions. No pooling hierarchies. Just attention over patches.

Patch embedding

Given an image of size $H \times W$ with $C$ channels, we divide it into a grid of non-overlapping patches, each of size $P \times P$. This produces $N = HW / P^2$ patches. For a 224×224 image with 16×16 patches, $N = 196$ patches. Each patch is flattened into a vector of dimension $P^2 \cdot C$ (for 16×16 RGB patches, this is $16 \times 16 \times 3 = 768$), then projected through a learned linear layer $E \in \mathbb{R}^{(P^2 \cdot C) \times d}$ into the model's hidden dimension $d$:

$$\mathbf{z}_i^0 = \mathbf{x}_i E + \mathbf{e}_i^{\text{pos}}$$

Here $\mathbf{x}_i$ is the flattened patch $i$ (a row vector in $\mathbb{R}^{P^2 C}$), $E$ is the patch embedding matrix, and $\mathbf{x}_i E$ is a matrix multiply ($\mathbb{R}^{1 \times P^2C} \times \mathbb{R}^{P^2C \times d} \to \mathbb{R}^{1 \times d}$) projecting the raw patch into the transformer's embedding space. $\mathbf{e}_i^{\text{pos}}$ is a learned positional embedding that tells the model where patch $i$ sits in the 2D grid.

Positional embeddings for 2D

Unlike text, image patches have two-dimensional positions. The original ViT paper explored several options: 1D learned positions (assign index 0 to N to patches in raster order, learn an embedding per index), 2D sinusoidal (separate sinusoidal encoding for row and column), and 2D learned. Surprisingly, learned 1D positions performed nearly as well as the 2D variants — the model learns to infer 2D structure from 1D ordering. A practical concern is resolution transfer: a model trained on 224×224 images (196 patches) cannot directly process 384×384 images (576 patches) because there are no learned embeddings for positions 197-576. The standard fix is bicubic interpolation of the position embedding grid to the new resolution, followed by fine-tuning.

The [CLS] token

Following BERT, ViT prepends a learnable [CLS] token to the patch sequence. This token has no spatial content — it attends to all patches and accumulates a global representation of the image. After the final transformer layer, the [CLS] token's output vector is fed to a classification MLP. An alternative is global average pooling over all patch outputs; both approaches work comparably, but [CLS] has become the convention because it mirrors the language model paradigm and simplifies downstream task heads.

The following code implements patch embedding with positional encoding in pure PyTorch.


import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    """Convert image into patch embeddings with positional encoding."""
    def __init__(self, img_size=224, patch_size=16, in_channels=3, d_model=768):
        super().__init__()
        self.patch_size = patch_size
        self.n_patches = (img_size // patch_size) ** 2  # 196 for 224/16

        # Linear projection: flatten patch → d_model
        self.proj = nn.Conv2d(
            in_channels, d_model,
            kernel_size=patch_size, stride=patch_size  # non-overlapping patches
        )  # output: (batch, d_model, H/P, W/P)

        # Learnable [CLS] token and position embeddings
        self.cls_token = nn.Parameter(torch.randn(1, 1, d_model))        # (1, 1, d_model)
        self.pos_embed = nn.Parameter(
            torch.randn(1, self.n_patches + 1, d_model)                  # (1, N+1, d_model)
        )

    def forward(self, x):
        # x: (batch, 3, 224, 224)
        B = x.shape[0]
        patches = self.proj(x)              # (batch, d_model, 14, 14)
        patches = patches.flatten(2)        # (batch, d_model, 196)
        patches = patches.transpose(1, 2)   # (batch, 196, d_model)

        # Prepend [CLS] token
        cls = self.cls_token.expand(B, -1, -1)   # (batch, 1, d_model)
        tokens = torch.cat([cls, patches], dim=1) # (batch, 197, d_model)

        # Add positional embeddings (element-wise addition, same shape)
        tokens = tokens + self.pos_embed           # (batch, 197, d_model)
        return tokens

# Example
embed = PatchEmbedding(img_size=224, patch_size=16, d_model=768)
img = torch.randn(2, 3, 224, 224)    # (batch=2, channels=3, H=224, W=224)
tokens = embed(img)                    # (2, 197, 768)
print(f"Output: {tokens.shape}")       # (2, 197, 768) — 196 patches + 1 CLS

Why ViT Works at Scale

The original ViT paper carried a surprising caveat: when trained on ImageNet alone (1.3M images), ViT-B performs worse than a comparable ResNet. The transformer lacks the inductive biases that make CNNs data-efficient — translation equivariance (the same filter detects a cat whether it appears top-left or bottom-right) and locality (nearby pixels are processed together). Without these priors, the model must learn spatial structure from scratch, and that requires far more data.

The data regime crossover

The picture changes completely at scale. When pre-trained on JFT-300M (300 million images), ViT-H/14 surpasses the best ResNet variant (ResNet152x4, pre-trained with the same data) on ImageNet. The advantage grows with dataset size: on JFT-300M, ViT shows no sign of saturation while the CNN plateaus. The intuition is clean: with enough data, the flexibility of self-attention — which can learn arbitrary spatial relationships, not just local ones — outweighs the efficiency of built-in spatial priors.

Scaling to 22 billion parameters

Google's ViT-22B (2023) demonstrated that vision transformers scale like language models. At 22 billion parameters, it required two key engineering innovations: QK normalization (normalizing query and key vectors before computing attention scores to prevent training instability) and asynchronous parallel linear operations (overlapping the QKV projection and feed-forward computation across devices). ViT-22B achieved state-of-the-art results across ImageNet, fairness benchmarks, and multimodal tasks, confirming that the scaling laws observed in language transformers transfer directly to vision.

The 2025 consensus: ViT vs CNN

By 2025 the debate has largely settled. ViTs dominate at scale: every major foundation model for vision (CLIP, SigLIP, DINOv2, DINOv3) uses a ViT backbone. CNNs remain competitive for edge deployment (smaller compute budgets, fixed-size inputs) and domains with limited data (medical imaging, industrial inspection). In practice, hybrid architectures like ConvNeXt (a CNN redesigned with transformer-era training recipes) and CoAtNet (CNN early stages + transformer late stages) often provide the best accuracy-compute tradeoff. The key lesson: ViT's advantage comes from its flexibility and scalability, not from any inherent architectural superiority on small problems.

DeiT: Data-Efficient Image Transformers

ViT's dependence on JFT-300M was a practical barrier — few researchers have access to 300 million labeled images. Touvron et al. (2021) showed that with the right training recipe, ViT can match CNN performance using only ImageNet-1k. The key ingredient is knowledge distillation from a strong CNN teacher.

The distillation token

DeiT introduces a second special token alongside [CLS]: a distillation token. While [CLS] is trained against the ground-truth label via cross-entropy, the distillation token is trained to match the output of a pre-trained CNN teacher (typically a RegNet). Both tokens attend to all patches and to each other, but they learn different things — the [CLS] token learns to classify directly, while the distillation token learns to mimic the teacher's behavior. At test time, the two predictions are averaged. This dual-token design outperforms standard knowledge distillation (which applies a single soft-target loss to the [CLS] output).

Strong augmentation and regularization

DeiT's training recipe is equally important. It combines RandAugment (random combinations of image transformations), MixUp (convex combinations of image pairs and their labels), CutMix (replacing a rectangular region of one image with another), random erasing, stochastic depth (randomly dropping transformer layers during training), and repeated augmentation (multiple augmented views of the same image in one batch). This heavy regularization compensates for ViT's lack of inductive biases, enabling it to learn from limited data without overfitting.

DeiT-III

DeiT-III (Touvron et al., 2022) refined the recipe further: a 3-augmentation strategy replacing the kitchen-sink approach, longer training schedules (400-800 epochs), and LayerScale (per-channel learnable scaling of each residual block's output). DeiT-III ViT-L/16 achieves 87.7% top-1 on ImageNet without any external data, competitive with models trained on far larger datasets. The lesson of DeiT is that architecture is only half the story — training methodology can close the gap that data volume was supposed to fill.

Self-Supervised ViT Pre-Training

Supervised pre-training on JFT-300M requires hundreds of millions of labeled images. Self-supervised methods eliminate this requirement entirely, learning powerful visual features from raw images with no human annotation. Two paradigms dominate: self-distillation (DINO) and masked image modeling (MAE). Each encodes a different inductive bias about what makes a good visual representation.

DINO: self-distillation with no labels

DINO (Caron et al., 2021) trains a student network and a teacher network on different augmented views of the same image. The teacher is not a separate pre-trained model — it is an exponential moving average (EMA) of the student's own weights. Two small crops and two large crops are generated from each image. The student sees all crops; the teacher sees only the large crops. The training objective is to match the student's output distribution to the teacher's:

$$\mathcal{L}_{\text{DINO}} = -\sum_{x \in \text{teacher crops}} \sum_{x' \in \text{student crops},\, x' \neq x} P_t(x) \log P_s(x')$$

Here $P_t(x)$ and $P_s(x')$ are probability distributions (vectors in $\mathbb{R}^K$ that sum to 1) obtained by passing the [CLS] output through a projection head and applying a softmax with temperature. The loss is a cross-entropy between the teacher's and student's distributions (both are vectors, and the summation is over the $K$ dimensions plus the crop pairs). Two tricks prevent collapse: centering (subtracting the running mean of the teacher's output) and sharpening (using a low temperature for the teacher's softmax).

The remarkable finding: DINO's self-attention maps spontaneously learn to segment foreground objects from background, without any segmentation labels or supervision. The [CLS] token's attention to individual patches forms a near-perfect saliency map. This emergent behavior suggests that the combination of multi-crop training and self-distillation forces the model to learn semantically meaningful spatial features.

iBOT: combining DINO with masked image modeling

iBOT (Zhou et al., 2022) extends DINO by adding a masked image modeling objective alongside the self-distillation loss. The student receives an image with some patches masked; it must predict the teacher's patch-level representations for the masked positions. This combines DINO's global self-distillation (on the [CLS] token) with local patch-level prediction, producing features that are strong for both image-level classification and dense prediction tasks like segmentation.

MAE: masked autoencoders

While DINO learns by matching views, MAE (He et al., 2022) learns by reconstruction. The idea is borrowed directly from masked language modeling in BERT: mask out a large fraction of input patches, encode the visible ones, then decode to reconstruct the masked patches in pixel space.

The critical design choice is the masking ratio: MAE masks 75% of patches — far more than BERT's 15% masking ratio for text. Why so much? Images are highly redundant: neighboring patches contain similar information, so masking 15% leaves enough context to reconstruct without understanding semantics. At 75% masking, the model must reason about high-level structure — object shapes, scene layout, semantic categories — to fill in the blanks.

The asymmetric encoder-decoder design is critical for efficiency. The encoder — which is the large, expensive ViT — processes only the 25% of visible patches. Learnable mask tokens are inserted only before the decoder, which is shallow (typically 8 transformer layers vs. 24 in the encoder). This means pre-training runs about 3-4× faster than encoding the full image.

Here is a simplified implementation of MAE's masking and encoding pipeline.


import torch
import torch.nn as nn

class MAEEncoder(nn.Module):
    """Simplified MAE: random masking + encoder on visible patches only."""
    def __init__(self, img_size=224, patch_size=16, d_model=768,
                 mask_ratio=0.75):
        super().__init__()
        self.n_patches = (img_size // patch_size) ** 2   # 196
        self.mask_ratio = mask_ratio
        self.n_visible = int(self.n_patches * (1 - mask_ratio))  # 49

        # Patch embedding (same as ViT)
        self.patch_embed = nn.Conv2d(
            3, d_model, kernel_size=patch_size, stride=patch_size
        )
        self.pos_embed = nn.Parameter(
            torch.randn(1, self.n_patches, d_model)      # (1, 196, d_model)
        )

        # Encoder: standard transformer layers (simplified as LayerNorm + identity)
        self.encoder = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(
                d_model=d_model, nhead=12, dim_feedforward=3072,
                batch_first=True, norm_first=True
            ),
            num_layers=12
        )

    def random_masking(self, x):
        """
        Random masking: keep only (1-mask_ratio) patches.
        Args:
            x: (batch, n_patches, d_model) — all patch embeddings
        Returns:
            visible: (batch, n_visible, d_model)
            mask: (batch, n_patches) — 0=visible, 1=masked
            ids_restore: (batch, n_patches) — for unshuffling later
        """
        B, N, D = x.shape
        n_vis = self.n_visible

        # Random permutation per sample
        noise = torch.rand(B, N, device=x.device)    # (batch, 196)
        ids_shuffle = noise.argsort(dim=1)            # (batch, 196)
        ids_restore = ids_shuffle.argsort(dim=1)      # (batch, 196)

        # Keep first n_vis patches (the ones with smallest random values)
        ids_keep = ids_shuffle[:, :n_vis]             # (batch, 49)
        visible = x.gather(
            1, ids_keep.unsqueeze(-1).expand(-1, -1, D)  # (batch, 49, d_model)
        )

        # Binary mask: 0=visible, 1=masked
        mask = torch.ones(B, N, device=x.device)      # (batch, 196)
        mask.scatter_(1, ids_keep, 0)                  # set visible positions to 0

        return visible, mask, ids_restore

    def forward(self, imgs):
        # imgs: (batch, 3, 224, 224)
        x = self.patch_embed(imgs)          # (batch, d_model, 14, 14)
        x = x.flatten(2).transpose(1, 2)    # (batch, 196, d_model)
        x = x + self.pos_embed              # (batch, 196, d_model)

        # Mask 75%, encode only visible 25%
        visible, mask, ids_restore = self.random_masking(x)
        # visible: (batch, 49, d_model) — only 25% of patches!

        encoded = self.encoder(visible)      # (batch, 49, d_model)
        return encoded, mask, ids_restore

# Example: encode only 49 out of 196 patches
mae = MAEEncoder(mask_ratio=0.75)
imgs = torch.randn(4, 3, 224, 224)         # (batch=4, 3, 224, 224)
encoded, mask, ids = mae(imgs)
print(f"Encoded: {encoded.shape}")           # (4, 49, 768) — 4x smaller than full
print(f"Masked: {mask.sum(1)[0].item():.0f} of {mask.shape[1]} patches")  # 147 of 196

I-JEPA: predicting in representation space

A philosophical critique of MAE: does predicting exact pixel values force the model to waste capacity on low-level texture? I-JEPA (Assran et al., 2023) tests this hypothesis by predicting masked patch representations in latent space rather than pixel values. A context encoder processes the visible patches; a predictor network maps from the context representations to predicted target representations; a target encoder (EMA of the context encoder, like DINO's teacher) provides the supervision signal. By operating in representation space, I-JEPA avoids the pressure to reconstruct fine-grained pixel detail and focuses on capturing semantic structure. Its successor V-JEPA 2 (Meta, 2025) extends this approach to video at massive scale, demonstrating that representation-space prediction scales as effectively as pixel-space reconstruction.

Key Insight

DINO and MAE represent two extremes of self-supervised ViT training. DINO operates entirely in representation space — it never reconstructs pixels. MAE operates entirely in pixel space — it reconstructs the raw image. I-JEPA occupies the middle ground: it uses masking like MAE but predicts in representation space like DINO. DINOv2 (and its successor DINOv3) combine both approaches, using DINO's self-distillation loss alongside iBOT's masked prediction loss.

DINOv2

DINO produces beautiful attention maps. MAE produces strong fine-tuning performance. The question was whether a single model could do both — and do both well enough to serve as a universal visual feature extractor, replacing task-specific pre-training entirely. DINOv2 (Oquab et al., 2023) answered yes.

Curated data at scale

DINOv2's first innovation is its training data. Rather than using raw internet scrapes, Meta built the LVD-142M dataset through a careful curation pipeline: starting from a large uncurated pool, they removed near-duplicates, filtered out memes and text-heavy images, and used self-supervised retrieval to balance the dataset across visual concepts. This curation proved critical — the same model trained on uncurated data performs significantly worse, confirming that data quality matters as much as data quantity for self-supervised learning.

Combined objectives

DINOv2 combines two losses: the DINO self-distillation loss (matching global [CLS] representations across views) and the iBOT masked prediction loss (predicting patch-level representations for masked tokens). This dual objective produces features that are simultaneously strong for image-level tasks (classification, retrieval) and dense prediction tasks (segmentation, depth estimation). A ViT-g/14 model with 1.1B parameters achieves state-of-the-art results across all these tasks with a frozen backbone — no fine-tuning needed, just a simple linear probe or lightweight head on top.

Register tokens

Darcet et al. (ICLR 2024) discovered a subtle artifact in large ViTs including DINOv2: certain patch tokens in low-information background regions develop anomalously high norms. These tokens appear to serve as internal "scratch pads" — the model repurposes them for global computations rather than representing local spatial content. This is harmless for classification (the [CLS] token still works fine) but corrupts dense prediction tasks that depend on per-patch features.

The fix is elegant: append a small number (typically 4) of extra learnable register tokens to the input sequence. These tokens have no spatial meaning — they exist solely to give the model a place to perform global computations without hijacking spatial patch tokens. After training, the register tokens are simply discarded for dense prediction tasks. DINOv2 models with registers are now the standard variant released by Meta.

DINOv3 and Beyond

DINOv2 established that a single self-supervised ViT could serve as a universal feature extractor. The 2025-2026 landscape extends this story in three directions: scaling the DINO recipe to its limits, showing that autoregressive pre-training works for vision too, and unifying vision and language training in a single encoder.

DINOv3

Meta released DINOv3 in August 2025 as the culmination of the DINO lineage. Three key innovations distinguish it from DINOv2. First, Gram Anchoring: a new loss term that regularizes the second-order correlations (Gram matrices) of patch features during training. Without this, dense feature quality degrades during long training schedules — a known but previously unsolved problem. Gram Anchoring explicitly prevents this degradation by penalizing drift in the correlation structure of patch representations.

Second, massive scale: DINOv3 trains a 7-billion-parameter ViT teacher on 1.7 billion images (vs. DINOv2's 142 million). Third, a high-resolution training stage specifically optimized for dense prediction tasks. The result: +6 mIoU over DINOv2 on ADE20K segmentation, 88.4% ImageNet top-1 (fine-tuned), and exceptional out-of-distribution robustness — 91.1% on ImageNet-R and 86.9% on ImageNet-Adversarial. Models from 21M to 7B parameters are all open-sourced.

AIMv2: autoregressive image modeling

Apple's AIMv2 (CVPR 2025 Highlight) demonstrates that vision transformers do not require masking or contrastive objectives at all. Instead, AIMv2 treats image patches as a sequence and predicts them autoregressively — left to right, top to bottom — exactly as a language model predicts the next token. A multimodal decoder jointly generates both raw patches and text tokens, unifying visual and linguistic representation learning. At 3 billion parameters with a frozen trunk, AIMv2 achieves 89.5% ImageNet top-1 and outperforms CLIP, SigLIP, and DINOv2 on several multimodal benchmarks. The message is clear: the ViT architecture is objective-agnostic — it works with contrastive, masked, and autoregressive training alike.

SigLIP 2: unified vision-language encoders

SigLIP 2 (Google DeepMind, February 2025) unifies multiple training objectives into a single recipe: sigmoid contrastive loss (matching images to text), LocCa decoder-based captioning, self-distillation, and masked prediction. Rather than choosing between contrastive and generative objectives, SigLIP 2 uses all of them. The result outperforms SigLIP at every model scale on zero-shot classification, retrieval, and as a vision-language model backbone. SigLIP 2 has become the default vision encoder for Google's PaliGemma 2 and other production VLMs, cementing the ViT's role as the universal vision backbone for the multimodal era.

Key Insight

The ViT architecture is remarkably objective-agnostic. The same transformer encoder produces strong visual features whether trained with contrastive learning (CLIP, SigLIP), self-distillation (DINO), masked reconstruction (MAE), representation prediction (I-JEPA), autoregressive generation (AIM), or combinations of all of these. This flexibility — inherited from the language transformer — is arguably ViT's most important property, more important than any specific architectural choice.
