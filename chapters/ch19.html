Chapter 19

Transformers for Detection and Segmentation

Object detection accumulated a decade of hand-engineered complexity: anchor boxes tuned per dataset, non-maximum suppression thresholds that silently dropped valid predictions, region proposal networks that added an entire subnetwork just to guess where objects might be. In 2020, DETR replaced all of it with a single transformer decoder and a matching loss — proving that detection could be formulated as set prediction. Within five years, that idea evolved into systems that detect objects described in free-form text, segment anything given a single click, and track every instance of a concept across video — all without anchors, NMS, or task-specific post-processing. By the end of this chapter you will understand how each of these systems works, why each architectural decision was made, and how the detection and segmentation lines converged into unified foundation models.

DETR: Detection as Set Prediction

Before DETR, every competitive object detector was a pipeline. Faster R-CNN generated region proposals, applied RoI pooling, ran a classifier, then filtered duplicates with non-maximum suppression. Each stage had hyperparameters — anchor aspect ratios, NMS IoU thresholds, score cutoffs — that were tuned per dataset and often per object category. The system worked, but it was fragile: change one hyperparameter and the entire downstream pipeline shifted.

DETR reframed detection as a direct set prediction problem. Given an image, predict a fixed-size set of $N$ detections (typically $N = 100$), where each detection is either an object (class label + bounding box) or a special "no object" token. Because the output is an unordered set, there is no need for anchors to define candidate locations, no need for NMS to remove duplicates, and no need for a region proposal stage at all.

Object Queries

The core mechanism is a set of $N$ learned embeddings called object queries. These are randomly initialized vectors in $\mathbb{R}^d$ that serve as "detection slots." Each query is fed into a transformer decoder, where it cross-attends to the encoder's output — a flattened feature map from a CNN backbone. Through this cross-attention, each query gathers spatial information from the image and specializes to predict a single object or "no object."

Think of object queries as $N$ detectives assigned to a crime scene. Each detective walks through the scene (cross-attends to image features), forms their own hypothesis about what happened where (predicts a class and box), and then the detectives compare notes with each other (self-attend among queries) to avoid two detectives reporting the same event.

Bipartite Matching Loss

The key question is: if the model outputs $N$ predictions and the image has $M$ objects (with $M \ll N$), how do you assign predictions to ground-truth objects for computing the loss? DETR uses the Hungarian algorithm for bipartite matching. We construct an $N \times M$ cost matrix where each entry measures how well prediction $i$ matches ground-truth object $j$, then find the minimum-cost one-to-one assignment:

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i=1}^{N} \mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)})$$

where $\sigma$ is a permutation and $\mathcal{L}_{\text{match}}$ combines classification probability and box distance. The matching cost for each pair is $-\hat{p}_{\sigma(i)}(c_i) + \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)})$ — the model wants high confidence on the correct class and a tight box fit. Predictions not matched to any ground-truth object are assigned to a "no object" class. The box loss itself is a combination of generalized IoU and L1 distance:

$$\mathcal{L}_{\text{box}} = \lambda_{\text{iou}}\, \mathcal{L}_{\text{GIoU}}(b_i, \hat{b}_{\sigma(i)}) + \lambda_{\text{L1}}\, \|b_i - \hat{b}_{\sigma(i)}\|_1$$

The GIoU term (generalized intersection over union, $\mathbb{R}^4 \times \mathbb{R}^4 \to \mathbb{R}$, a scalar similarity between two bounding boxes) handles scale invariance, while the L1 term provides a direct regression signal. Because matching is one-to-one, each ground-truth object gets exactly one prediction — no duplicates, no NMS needed.

DETR Limitations

DETR's elegance came at a cost. Training required 500 epochs on COCO — roughly 10-20x longer than Faster R-CNN. The global attention in the encoder computed similarity between every pair of spatial positions, which was expensive at high resolutions and diluted fine-grained spatial information needed for small objects. DETR's AP on small objects lagged significantly behind Faster R-CNN. These limitations set the agenda for the next generation of detection transformers.

The following code demonstrates a simplified DETR decoder with bipartite matching loss.


import torch
import torch.nn as nn
from scipy.optimize import linear_sum_assignment

class SimpleDETRDecoder(nn.Module):
    def __init__(self, d_model=256, n_queries=100, n_classes=80):
        super().__init__()
        self.queries = nn.Embedding(n_queries, d_model)  # learned object queries
        decoder_layer = nn.TransformerDecoderLayer(d_model, nhead=8, batch_first=True)
        self.decoder = nn.TransformerDecoder(decoder_layer, num_layers=6)
        self.class_head = nn.Linear(d_model, n_classes + 1)  # +1 for "no object"
        self.box_head = nn.Sequential(
            nn.Linear(d_model, d_model), nn.ReLU(),
            nn.Linear(d_model, 4)  # (cx, cy, w, h) normalized
        )

    def forward(self, encoder_memory):
        # encoder_memory: (batch, hw, d_model) — flattened CNN features
        B = encoder_memory.size(0)
        q = self.queries.weight.unsqueeze(0).expand(B, -1, -1)  # (batch, n_queries, d_model)
        out = self.decoder(q, encoder_memory)         # (batch, n_queries, d_model)
        logits = self.class_head(out)                  # (batch, n_queries, n_classes+1)
        boxes = self.box_head(out).sigmoid()           # (batch, n_queries, 4)
        return logits, boxes

def hungarian_loss(pred_logits, pred_boxes, gt_classes, gt_boxes):
    # pred_logits: (n_queries, n_classes+1), pred_boxes: (n_queries, 4)
    # gt_classes: (m,), gt_boxes: (m, 4)
    n = pred_logits.size(0)
    m = gt_classes.size(0)
    # Cost matrix: classification cost + L1 box cost
    prob = pred_logits.softmax(-1)                     # (n_queries, n_classes+1)
    cls_cost = -prob[:, gt_classes]                     # (n_queries, m)
    box_cost = torch.cdist(pred_boxes, gt_boxes, p=1)  # (n_queries, m) — L1 distance
    cost = cls_cost + 5.0 * box_cost                   # combined matching cost
    row_idx, col_idx = linear_sum_assignment(cost.detach().cpu().numpy())
    # Compute loss on matched pairs
    loss_cls = nn.functional.cross_entropy(pred_logits[row_idx], gt_classes[col_idx])
    loss_box = nn.functional.l1_loss(pred_boxes[row_idx], gt_boxes[col_idx])
    return loss_cls + 5.0 * loss_box

Deformable DETR: Fixing Attention Cost

DETR's encoder computed global self-attention over every spatial position in the feature map. For a typical COCO image with a feature map of $H \times W = 25 \times 34 = 850$ positions, that meant $850^2 \approx 722{,}000$ attention pairs — per head, per layer. At higher resolutions the cost became prohibitive, and the model spent most of its attention budget on uninformative background-to-background pairs.

Deformable DETR replaced global attention with deformable attention: instead of attending to all spatial positions, each query learns a small set of $K$ reference points (typically $K = 4$) and samples features at those locations using bilinear interpolation. The attention is sparse by construction — it looks only where it needs to.

Multi-Scale Deformable Attention

The real power comes from extending deformable attention across multiple feature scales. Each query samples $K$ reference points from each of $L$ feature pyramid levels, giving it access to both high-resolution details (for small objects) and low-resolution context (for large objects). The total cost per query is $O(LKC)$ — linear in spatial resolution rather than quadratic. Formally:

$$\text{DeformAttn}(z_q, \hat{p}_q, \{x^l\}_{l=1}^{L}) = \sum_{m=1}^{M} W_m \left[\sum_{l=1}^{L} \sum_{k=1}^{K} A_{mlqk} \cdot x^l(\hat{p}_q + \Delta p_{mlqk})\right]$$

where $z_q$ is the query feature, $\hat{p}_q$ is its reference point, $A_{mlqk}$ are learned attention weights (scalars, sum to 1 across all $l$ and $k$ for each head $m$; element-wise scaling of the sampled feature), $\Delta p_{mlqk}$ are learned sampling offsets, $x^l(\cdot)$ denotes bilinear interpolation from feature level $l$, and $W_m$ is the per-head projection (matrix multiply, $W_m \in \mathbb{R}^{d \times d_v}$, projecting the $d_v$-dimensional aggregated value to the output space). The result: 10x faster convergence (50 epochs matches DETR at 500) and significantly better small-object detection.

The DETR Improvement Arc

Between 2022 and 2026, a rapid succession of papers refined the DETR paradigm. Each addressed a specific bottleneck — training signal sparsity, convergence speed, or inference latency — while preserving the core set-prediction idea. The result is a family of detectors that pushed COCO AP from DETR's original 42.0 to over 60, with latencies competitive with YOLO.

DN-DETR: Denoising Training

The one-to-one Hungarian matching in DETR produces a sparse training signal: each decoder query gets supervision from at most one ground-truth object, and most queries are assigned to "no object." DN-DETR supplements this sparse signal with a denoising task. During training, the model adds Gaussian noise to ground-truth bounding boxes — $\tilde{b} = b + \epsilon$, where $\epsilon \sim \mathcal{N}(0, \lambda^2 I)$ (element-wise addition of noise to each box coordinate, both vectors in $\mathbb{R}^4$) — and feeds these noisy boxes as additional decoder queries. The decoder must reconstruct the clean boxes from the noisy inputs. This gives every ground-truth object a dense, direct supervision signal beyond the single matched query, and convergence improves substantially.

Co-DETR: Collaborative Hybrid Assignment

Co-DETR attacked the sparsity problem from a different angle. The insight: one-to-one matching is essential at inference (to avoid duplicates), but during training, the one-to-many assignment strategy from classical detectors like ATSS produces denser gradients. Co-DETR adds auxiliary decoder heads that use one-to-many matching — each ground-truth object can supervise multiple predictions. These auxiliary heads share the encoder with the primary one-to-one decoder, enriching encoder gradients without affecting inference cost (the auxiliary heads are discarded at test time). The combined loss is:

$$\mathcal{L} = \mathcal{L}_{\text{one-to-one}} + \sum_{k=1}^{K} \alpha_k \, \mathcal{L}_{\text{one-to-many}}^{(k)}$$

where each $\alpha_k$ is a scalar weight for auxiliary head $k$ (scalar multiplication). Co-DETR achieved state-of-the-art results on LVIS, the large-vocabulary detection benchmark with 1,200+ categories.

DINO (Detection)

DINO — the detection model, distinct from the self-supervised method discussed in Chapter 17 — combined denoising training with contrastive denoising. It generates both positive queries (slightly noised GT boxes that should reconstruct the original) and negative queries (heavily noised boxes that should predict "no object"). This contrastive signal sharpens the boundary between "close enough to be a match" and "too far to be valid." Combined with a mixed query selection strategy that initializes some object queries from encoder features rather than from scratch, DINO pushed COCO detection to 63.3 AP with a Swin-L backbone.

RT-DETR: Real-Time Detection

RT-DETR demonstrated that detection transformers could be competitive with YOLO on latency. It replaced the heavyweight encoder-decoder stack with an efficient hybrid encoder that combines CNN-based intra-scale feature extraction with transformer-based cross-scale feature fusion. By also optimizing the decoder and removing NMS entirely, RT-DETR achieved real-time inference while maintaining strong accuracy — the first detection transformer to close the speed gap with one-stage detectors.

RF-DETR: Neural Architecture Search for Detection

RF-DETR, published at ICLR 2026 by Roboflow, took the real-time line further. It uses a frozen DINOv2 ViT backbone — directly leveraging the powerful self-supervised features from Chapter 18 — and applies Neural Architecture Search (NAS) to find the optimal encoder configuration. The search explores thousands of configurations with shared weights, ultimately selecting an architecture that alternates windowed and global attention modules to balance local detail with global context. RF-DETR surpasses 60 mAP on COCO with latencies under 40ms and includes a lightweight segmentation head for instance masks, blurring the traditional boundary between detection and segmentation.

Open-Vocabulary and Unified Detection

Every detector discussed so far operates on a fixed vocabulary: the model is trained on COCO's 80 categories or LVIS's 1,200 and can only detect those. But real-world applications need to find objects described in free-form text — "the red coffee mug next to the keyboard" — including objects the model has never seen during training. This requires conditioning detection on language.

Grounding DINO

Grounding DINO fuses a text encoder (BERT) with the DINO detection decoder. Text tokens and image features exchange information through bidirectional cross-attention: image features attend to text tokens (to understand what to look for) and text tokens attend to image features (to ground language in visual evidence). At inference, the user provides a text prompt — "all red cars" — and the model localizes matching objects with bounding boxes. The model was trained with ECCV 2024 publication on phrase grounding data, converting closed-set detection into open-vocabulary grounding.

The following code demonstrates the core mechanism — bidirectional cross-attention between text and image features.


import torch
import torch.nn as nn

class BidirectionalCrossAttention(nn.Module):
    """Grounding DINO-style cross-attention between image and text features."""
    def __init__(self, d_model=256, n_heads=8):
        super().__init__()
        self.img_to_text = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.text_to_img = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm_img = nn.LayerNorm(d_model)
        self.norm_text = nn.LayerNorm(d_model)

    def forward(self, img_feats, text_feats):
        # img_feats:  (batch, hw, d_model) — flattened spatial features
        # text_feats: (batch, n_tokens, d_model) — encoded text prompt
        # Image queries attend to text keys/values
        img_cross, _ = self.img_to_text(
            query=img_feats, key=text_feats, value=text_feats
        )  # (batch, hw, d_model) — image features enriched with text context
        img_out = self.norm_img(img_feats + img_cross)

        # Text queries attend to image keys/values
        text_cross, _ = self.text_to_img(
            query=text_feats, key=img_feats, value=img_feats
        )  # (batch, n_tokens, d_model) — text features grounded in visual evidence
        text_out = self.norm_text(text_feats + text_cross)
        return img_out, text_out

# Usage: fuse language understanding with spatial features
cross_attn = BidirectionalCrossAttention(d_model=256)
img = torch.randn(2, 850, 256)   # (batch, H*W, d_model)
text = torch.randn(2, 12, 256)    # (batch, n_tokens, d_model)
img_enhanced, text_grounded = cross_attn(img, text)

DINO-X: Universal Object-Level Vision

DINO-X, from IDEA Research, scaled the Grounding DINO architecture into a unified vision model for open-world understanding. Trained on Grounding-100M — over 100 million human-annotated grounding samples — DINO-X supports detection, segmentation, phrase grounding, visual-prompt counting, pose estimation, and region captioning through a single model. It also introduces a universal object prompt that enables prompt-free detection: the model can detect everything in an image without any user-specified text. On COCO zero-shot (no COCO training data), DINO-X achieves 56.0 AP — demonstrating that scale and diverse task heads can produce a general-purpose object-level vision system.

Florence-2: Vision as Sequence-to-Sequence

Florence-2, from Microsoft, took a more radical approach. Instead of adding task-specific heads, it formulates every vision task as a sequence-to-sequence problem. The input is an image plus a text prompt ("detect all objects," "segment the dog," "describe the scene"), and the output is a text sequence that encodes bounding boxes, segmentation masks, or captions as structured tokens. The architecture is a DaViT vision encoder combined with a BERT text encoder feeding into a standard autoregressive transformer decoder. At only 0.2B-0.7B parameters, Florence-2 achieves results competitive with much larger models — proving that you do not need separate heads per task if you can express all outputs as token sequences.

Segmentation Transformers

Detection finds bounding boxes. Segmentation goes further — it classifies every pixel. The challenge is that segmentation comes in three flavors: semantic segmentation (label every pixel with a class, but do not distinguish instances), instance segmentation (identify each object instance with its own mask), and panoptic segmentation (both: every pixel gets a class label and an instance ID for "things," and just a class label for "stuff" like sky or road). Historically, each required a different architecture. Transformer-based approaches unified them.

SegFormer

SegFormer pairs a hierarchical transformer encoder (the Mix Transformer, or MiT) with an extremely lightweight all-MLP decoder. The encoder produces multi-scale features at 1/4, 1/8, 1/16, and 1/32 resolution — similar to the hierarchical ViTs from Chapter 18 — using overlapping patch embeddings instead of positional encodings. The decoder simply upsamples features from all scales to 1/4 resolution, concatenates them, and passes through two MLPs:

$$\hat{y} = \text{MLP}\!\left(\text{Concat}(\text{Up}(F_1), \text{Up}(F_2), \text{Up}(F_3), \text{Up}(F_4))\right)$$

where each $F_i$ is the feature map at scale $i$ (matrix multiply inside the MLP: $W F_{\text{concat}} + b$, with $W \in \mathbb{R}^{C_{\text{out}} \times 4C}$). The simplicity is the point: if the encoder features are good enough, the decoder barely needs to do any work. SegFormer achieved strong results on ADE20K and Cityscapes while being significantly faster than prior approaches.

Mask2Former: Universal Segmentation

Mask2Former unified all three segmentation tasks into a single architecture. The core insight: all segmentation tasks are fundamentally mask prediction tasks — the only difference is how you aggregate the masks at the end. The architecture uses a pixel decoder to produce multi-scale features, then a transformer decoder where each query predicts one mask. The critical mechanism is masked cross-attention: each query attends only to the spatial region predicted by its mask from the previous decoder layer, not the entire feature map:

$$\text{MaskedAttn}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}} + M_{\ell-1}\right)V$$

where $QK^\top$ is the attention score matrix (matrix multiply, $Q \in \mathbb{R}^{N \times d_k}$, $K \in \mathbb{R}^{HW \times d_k}$, yielding $\mathbb{R}^{N \times HW}$), and $M_{\ell-1}$ is a mask that is $-\infty$ at locations outside the predicted mask from layer $\ell - 1$ (element-wise addition, $M_{\ell-1} \in \mathbb{R}^{N \times HW}$). This constrains each query to refine its own spatial region rather than competing over the entire feature map. The same architecture handles semantic segmentation (57.7 mIoU on ADE20K), instance segmentation (50.1 AP on COCO), and panoptic segmentation (57.8 PQ on COCO) by changing only the loss and post-processing.

SegGPT: In-Context Segmentation

SegGPT treats segmentation as an in-context learning problem, analogous to few-shot prompting in language models. Given a reference image paired with its segmentation mask (the "context"), the model segments the same concept in a new query image. During training, SegGPT uses a clever proxy task: randomly assign colors to each segment in an image, then train the model to reproduce that coloring. Because the colors are random, the model cannot memorize category-to-color mappings — it must learn to segment based on visual similarity alone. This enables zero-shot transfer to novel object categories through visual prompting.

Segment Anything: Promptable Segmentation

The segmentation architectures above still require training data with category labels. SAM (Segment Anything Model) asked a different question: can you build a single model that segments any object in any image, given only a spatial prompt — a click, a bounding box, or a rough mask? The answer required not just a new architecture but a new dataset: SA-1B, containing over 1 billion masks on 11 million images, generated through a human-in-the-loop data engine where SAM's own predictions were iteratively refined by human annotators.

SAM Architecture

SAM has three components. The image encoder is a heavyweight ViT-H (632M parameters) that runs once per image and produces dense feature maps. The prompt encoder converts spatial prompts — points, boxes, or masks — into token embeddings using learned positional encodings for points/boxes and convolutions for mask prompts. The mask decoder is a lightweight two-layer transformer decoder (only ~4M parameters) that takes prompt tokens as queries, cross-attends to image features, and outputs mask logits through a dot product:

$$M = \text{Sigmoid}(q_{\text{out}}^\top \text{Up}(f_{\text{img}}))$$

where $q_{\text{out}}^\top \text{Up}(f_{\text{img}})$ is a dot product between the decoder output token $q_{\text{out}} \in \mathbb{R}^{C}$ and each spatial position in the upsampled image features $\text{Up}(f_{\text{img}}) \in \mathbb{R}^{C \times H \times W}$ (dot product over the channel dimension, yielding a scalar per spatial position, resulting in a mask of shape $\mathbb{R}^{H \times W}$). The image encoder runs once; the mask decoder runs per prompt in milliseconds. This asymmetry is deliberate — it enables interactive use where the user clicks multiple times on the same image.

The following code demonstrates SAM's lightweight mask decoder.


import torch
import torch.nn as nn

class SAMStyleMaskDecoder(nn.Module):
    """Simplified SAM mask decoder: 2-layer transformer + dot-product mask."""
    def __init__(self, d_model=256, n_heads=8, n_mask_tokens=3):
        super().__init__()
        # Two transformer decoder layers (lightweight)
        decoder_layer = nn.TransformerDecoderLayer(d_model, n_heads, batch_first=True)
        self.decoder = nn.TransformerDecoder(decoder_layer, num_layers=2)
        # Mask output tokens (predict multiple masks for ambiguity)
        self.mask_tokens = nn.Embedding(n_mask_tokens, d_model)
        # Upsampling: 16x16 → 64x64 feature map
        self.upsample = nn.Sequential(
            nn.ConvTranspose2d(d_model, d_model // 4, kernel_size=2, stride=2),
            nn.GELU(),
            nn.ConvTranspose2d(d_model // 4, d_model // 8, kernel_size=2, stride=2),
        )
        self.mask_proj = nn.Linear(d_model, d_model // 8)

    def forward(self, img_embed, prompt_tokens):
        # img_embed:     (batch, hw, d_model) — from frozen image encoder
        # prompt_tokens: (batch, n_prompts, d_model) — encoded clicks/boxes
        B, hw, C = img_embed.shape
        H = W = int(hw ** 0.5)
        mask_tok = self.mask_tokens.weight.unsqueeze(0).expand(B, -1, -1)
        queries = torch.cat([mask_tok, prompt_tokens], dim=1)  # (batch, n_mask+n_prompts, d_model)
        out = self.decoder(queries, img_embed)                 # (batch, n_mask+n_prompts, d_model)
        mask_out = out[:, :self.mask_tokens.num_embeddings]    # (batch, n_mask, d_model)
        # Reshape image features and upsample
        img_2d = img_embed.transpose(1, 2).view(B, C, H, W)   # (batch, d_model, H, W)
        img_up = self.upsample(img_2d)                         # (batch, d_model//8, 4H, 4W)
        # Dot product: mask tokens × upsampled features → spatial masks
        mask_out = self.mask_proj(mask_out)                    # (batch, n_mask, d_model//8)
        masks = torch.einsum('bmc,bchw->bmhw', mask_out, img_up)  # (batch, n_mask, 4H, 4W)
        return masks.sigmoid()

SAM 2: Extending to Video

SAM 2 extended SAM from images to video with three additions. A memory encoder compresses the model's predictions on each frame into compact memory tokens. A memory bank stores these tokens from recent frames (typically the last 6-8 frames plus any user-prompted frames). And memory attention lets the current frame's features attend to stored memories, enabling temporal consistency — if the model segmented a dog in frame 10, it can recall that information when processing frame 15:

$$\text{MemAttn}(z_t, M_{1:t-1}) = \text{Attention}(z_t,\; [z_t;\, M_{1:t-1}],\; [z_t;\, M_{1:t-1}])$$

where $z_t$ are the current frame's features and $M_{1:t-1}$ are stored memory tokens (matrix multiply inside the attention: $z_t [z_t; M_{1:t-1}]^\top$ computes attention scores, $\mathbb{R}^{HW \times d} \times \mathbb{R}^{d \times (HW + |\text{mem}|)} \to \mathbb{R}^{HW \times (HW + |\text{mem}|)}$). The user can provide prompts on any frame — click on frame 1 to select a person, and SAM 2 tracks them through the entire video at 44 FPS.

Efficient-SAM2: Post-Training Acceleration

Efficient-SAM2, published at ICLR 2026, accelerates SAM 2 without retraining the model. The authors observed that SAM 2 exhibits sparse perception patterns: the mask decoder's attention concentrates on foreground objects, and only a small subset of memory tokens contribute meaningfully to memory attention. Efficient-SAM2 exploits this with two techniques. Sparse Window Routing (SWR) in the image encoder routes only windows containing objects through the full ViT — background windows get a cheaper approximation. Sparse Memory Retrieval (SMR) in memory attention selects only the most relevant memory tokens per query instead of attending to all stored memories. Together, these achieve a 1.68x speedup on SAM 2.1-L with only 1.0% accuracy drop on the SA-V benchmark.

SAM 3: Segment Anything with Concepts

SAM and SAM 2 require the user to point at every instance they want to segment. SAM 3, presented at ICLR 2026, introduced concept-prompted segmentation: instead of pointing, the user specifies a concept — a short noun phrase like "yellow school bus," an image exemplar, or both — and SAM 3 finds and segments all instances of that concept in the image or video. The architecture uses a dual encoder-decoder transformer: one path processes the concept prompt, the other processes the image, with cross-attention for alignment. A dedicated presence head predicts whether a concept exists in the frame before attempting localization, improving detection accuracy. SAM 3 was trained on the new SA-Co dataset (4M unique concept labels with hard negatives) and evaluated on the SA-Co benchmark (214K unique concepts, 124K images, 1.7K videos), where it doubled the accuracy of prior concept-segmentation systems.

Grounded SAM 2: Detection Meets Segmentation

Grounded SAM 2 is not a single model but a pipeline that ties the detection and segmentation stories together. An open-vocabulary detector — Grounding DINO, DINO-X, or Florence-2 — takes a text prompt and produces bounding boxes. These boxes serve as spatial prompts to SAM 2, which generates precise masks and tracks them across video frames. The result is an end-to-end system for open-set detection, segmentation, and tracking: describe what you want in natural language, and get pixel-precise, temporally consistent masks throughout a video. This pipeline represents the practical convergence of the detection and segmentation lines traced throughout this chapter.

DINO and DINOv2 for Dense Prediction

The most striking demonstration of transformers for dense prediction requires no detection or segmentation architecture at all. The self-supervised DINO and DINOv2 features from Chapter 17 turn out to contain rich spatial information that was never explicitly trained for.

Emergent Segmentation from Attention Maps

When you visualize the CLS token's attention map from a DINO ViT, the attention spontaneously segments foreground objects from background — without any segmentation training. The self-distillation objective forces the student to match the teacher's representation, which implicitly requires understanding object boundaries. These attention maps can be used directly for unsupervised object discovery, or as pseudo-labels for training segmentation models without manual annotation.

DINOv2 + Linear Probes

DINOv2's features are even more structured. A frozen DINOv2 ViT-g backbone with a single linear layer achieves strong results on monocular depth estimation, semantic segmentation, and surface normal estimation. The features are so well-organized that a linear probe suffices — no fine-tuning, no complex decoder. This connects back to Chapter 18's central lesson: when pretraining is good enough, the architecture of the downstream head matters less and less. The features encode 3D structure, semantic categories, and fine spatial detail without ever being asked to.

Depth Pro: Zero-Shot Metric Depth

Depth Pro, from Apple, demonstrates this principle at its most extreme. It uses DINOv2 features from multiple scales, processing overlapping patches at different resolutions with a shared ViT encoder. A global context branch captures the overall scene structure (for metric scale estimation), while the patch branches capture fine spatial detail. A lightweight DPT-style decoder combines these into sharp, metric depth maps — meaning the predicted depths have real-world scale, not just relative ordering. Depth Pro produces 2.25-megapixel depth maps in 0.3 seconds on a single GPU, without requiring camera intrinsics or any metadata. The 3D understanding was never supervised — it emerged from the DINOv2 pretraining.

Key Insight

The convergence story of this chapter runs deeper than architecture. Detection evolved from hand-engineered pipelines to clean set prediction. Segmentation evolved from task-specific heads to universal promptable systems. And self-supervised pretraining quietly learned both detection and segmentation features without being asked to. The trajectory points toward a future where the task-specific head becomes trivial — perhaps just a linear layer — and all the intelligence lives in the foundation model's features.
