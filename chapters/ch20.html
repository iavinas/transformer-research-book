Chapter 20

Image Generation with Transformers

In 2021, DALL-E generated images at 256×256 by predicting discrete tokens one at a time — a slow, autoregressive process that often produced spatially incoherent results. By early 2026, a 0.6-billion-parameter model generates photorealistic 1024×1024 images in under one second on a laptop GPU, and a 32-billion-parameter model produces 4K images with perfectly readable text. The transformation happened in two stages: first, diffusion models replaced autoregressive token prediction; then, transformers replaced the UNet as the diffusion backbone. This chapter traces both transitions, from DALL-E's discrete VAE tokens through latent diffusion's continuous latents to the Diffusion Transformer (DiT), the multimodal dual-stream MMDiT, and the rectified flow formulations that power today's frontier models. Every architectural concept here — DiT, AdaLN-Zero, MMDiT, ControlNet, classifier-free guidance — reappears in the next two chapters on video, making this chapter the foundation for all generative vision transformers.

The Image Generation Revolution

Before diffusion models, the dominant paradigm for high-quality image generation was GANs — generative adversarial networks that trained a generator and discriminator in opposition. GANs produced sharp images but suffered from mode collapse (generating only a subset of the data distribution), training instability, and difficulty scaling to diverse, open-domain generation. Diffusion models solved these problems by framing generation as iterative denoising: start from pure Gaussian noise and progressively remove noise over many steps until a clean image emerges.

The key insight that made diffusion practical for high-resolution images was latent diffusion: instead of denoising in pixel space (expensive at high resolutions), first compress the image into a compact latent representation using a pretrained autoencoder, then run the diffusion process in this latent space. The autoencoder handles spatial detail; the diffusion model handles semantic structure. This separation of concerns reduced the computational cost by orders of magnitude while maintaining image quality.

The autoencoder comes in two flavors. A VQ-VAE (vector-quantized variational autoencoder) maps images to discrete codebook indices — each spatial position becomes one of $K$ discrete tokens. A KL-VAE maps images to continuous latent vectors with a KL divergence regularizer. DALL-E used VQ-VAE; Stable Diffusion and all modern diffusion models use KL-VAE because continuous latents are more compatible with the Gaussian noise process in diffusion.

Autoregressive Image Generation

Before diffusion transformers, the transformer's role in image generation was autoregressive: tokenize the image into discrete codes, then predict them one at a time, left to right, top to bottom — exactly like a language model predicts text tokens.

DALL-E

DALL-E (OpenAI, 2021) used a two-stage approach. First, a discrete VAE (dVAE) was trained to compress 256×256 images into a 32×32 grid of discrete tokens from a codebook of 8,192 entries. Second, a 12-billion-parameter autoregressive transformer was trained to model the joint distribution over 256 text tokens and 1,024 image tokens (32×32), predicting each image token conditioned on all preceding tokens and the text. At inference, the transformer generates image tokens autoregressively, then the dVAE decoder converts them to pixels.

Parti (Google, 2022) scaled this approach to 20 billion parameters and used a ViT-VQGAN tokenizer for sharper reconstructions. The results demonstrated that autoregressive image generation improves reliably with scale — but the fundamental limitation remained: generating 1,024 tokens sequentially is slow, and errors in early tokens propagate to all subsequent ones. A misplaced edge in token 50 corrupts the spatial structure of the remaining 974 tokens.

Diffusion models solved this by generating all spatial positions simultaneously. Instead of sequential token prediction, diffusion starts from noise everywhere and refines the entire image in parallel across each denoising step. The transformer's role shifted from autoregressive predictor to denoising backbone.

Diffusion Transformers (DiT)

The Stable Diffusion era (2022-2023) used a UNet as the denoising backbone. UNets worked well — their skip connections and multi-scale feature maps were naturally suited to the denoising task — but they were architecturally complex and did not scale as predictably as transformers.

DiT: Replacing UNet with a Transformer

DiT (Peebles and Xie, 2022) showed that a standard vision transformer, with one key modification, could replace the UNet entirely. The input noisy latent $z_t \in \mathbb{R}^{h \times w \times c}$ is patchified into tokens (exactly like ViT), and the transformer predicts the noise $\epsilon$ to subtract. The key modification is AdaLN-Zero — adaptive layer normalization that conditions each layer on the diffusion timestep $t$ and text embedding $c$:

$$\text{AdaLN}(h, t, c) = \gamma(t, c) \odot \frac{h - \mu}{\sigma} + \beta(t, c)$$

where $\gamma(t, c), \beta(t, c) \in \mathbb{R}^d$ are scale and shift vectors predicted by a small MLP from the combined timestep and text embeddings (element-wise product $\odot$ between $\gamma$ and the normalized activation, both in $\mathbb{R}^d$). An additional gate parameter $\alpha(t, c)$ is initialized to zero, so the transformer starts as an identity function and gradually learns the denoising task — hence "AdaLN-Zero."

DiT demonstrated a clean scaling law: FID (Fréchet Inception Distance, a measure of image quality) improved log-linearly with model compute. Doubling the parameters consistently improved results, with no sign of diminishing returns. This scaling predictability — inherited from the transformer architecture — was the argument for replacing UNets.

The following code demonstrates a complete DiT block with AdaLN-Zero conditioning.


import torch
import torch.nn as nn

class DiTBlock(nn.Module):
    """Diffusion Transformer block with AdaLN-Zero conditioning."""
    def __init__(self, d_model=1152, n_heads=16):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm2 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.ffn = nn.Sequential(nn.Linear(d_model, 4 * d_model), nn.GELU(),
                                 nn.Linear(4 * d_model, d_model))
        # AdaLN-Zero: predict (gamma1, beta1, alpha1, gamma2, beta2, alpha2)
        self.adaln_mlp = nn.Sequential(nn.SiLU(), nn.Linear(d_model, 6 * d_model))

    def forward(self, x, cond):
        # x: (batch, n_patches, d_model) — patchified noisy latent
        # cond: (batch, d_model) — timestep embedding + text pooled embedding
        params = self.adaln_mlp(cond).unsqueeze(1)              # (B, 1, 6*d)
        g1, b1, a1, g2, b2, a2 = params.chunk(6, dim=-1)       # each (B, 1, d)
        # Modulated attention with zero-initialized gate
        h = g1 * self.norm1(x) + b1                             # (B, N, d) element-wise
        attn_out, _ = self.attn(h, h, h)                        # (B, N, d)
        x = x + a1 * attn_out                                   # gated residual (a1 starts at 0)
        # Modulated FFN with zero-initialized gate
        h = g2 * self.norm2(x) + b2
        x = x + a2 * self.ffn(h)
        return x  # (batch, n_patches, d_model)

MMDiT and Rectified Flow

DiT conditioned on text through adaptive layer normalization — effective but limited. The text signal modulated every layer equally, without the fine-grained text-image interaction that cross-attention in UNets provided. MMDiT (Multimodal Diffusion Transformer), introduced in Stable Diffusion 3 and FLUX, solved this with a fundamentally different design.

Dual-Stream Joint Attention

MMDiT processes text and image tokens as two separate streams. Each stream has its own MLP (feed-forward network) that processes modality-specific features. But at the attention step, the two streams are concatenated into a single sequence and processed by full self-attention — enabling bidirectional information flow. The text tokens can attend to image tokens (to ground language in visual content) and image tokens can attend to text tokens (to understand what to generate). After attention, the streams are split back for modality-specific MLP processing.

This dual-stream design avoids the one-directional limitation of cross-attention (where text informs image but image cannot inform text understanding) while maintaining modality-specific feature processing through separate MLPs.

Rectified Flow

Alongside the architectural shift, the training formulation changed. Standard diffusion models learn to denoise along curved trajectories — the path from noise to image follows a complex, curved schedule. Rectified flow learns straight-line paths from noise to data. The training objective becomes simpler: given a noise sample $\epsilon$ and a data sample $x_0$, the model learns to predict the velocity $v$ along the straight line connecting them:

$$v_\theta(z_t, t) = x_0 - \epsilon$$

where $z_t = (1-t) \cdot \epsilon + t \cdot x_0$ is the linear interpolation at time $t \in [0, 1]$ (element-wise scalar-vector multiplication, with $t$ scaling the clean image and $(1-t)$ scaling the noise, both in $\mathbb{R}^{h \times w \times c}$). Straight paths require fewer sampling steps — FLUX generates high-quality images in 12-20 steps, compared to 50+ for standard diffusion. This efficiency gain was critical for making large models practical.

Text Encoding Strategies

Modern image generation models use powerful text encoders to convert prompts into conditioning signals. FLUX uses T5-XXL (4.7B parameters), providing rich semantic understanding and the ability to render readable text in images. Stable Diffusion 3 uses dual encoders — CLIP for alignment and T5 for text understanding. The quality of the text encoder directly limits prompt adherence: models with T5-XXL consistently outperform those with CLIP-only conditioning on complex, multi-object prompts.

The Frontier of Image Generation (2025-2026)

By early 2026, the DiT+MMDiT+rectified flow combination has become the standard architecture. The competition is now on scale, data quality, and training recipes.

FLUX 2

FLUX 2 (Black Forest Labs, November 2025) is a 32-billion-parameter rectified flow transformer that produces 4K-class (4 megapixel) photorealistic images in 12-20 steps. Its architecture combines multimodal DiT blocks with a Vision-Language Model integration for superior prompt understanding. FLUX 2 renders perfectly readable text in generated images — a capability that eluded all previous models — thanks to the T5-XXL text encoder's deep understanding of character-level semantics.

Stable Diffusion 3.5

Stable Diffusion 3.5 (Stability AI) uses the MMDiT-X architecture at 8 billion parameters, delivering 1-megapixel images with strong prompt adherence. It also introduced adversarial diffusion distillation (ADD) for the Large Turbo variant, which reduces sampling to just 1-4 steps while maintaining quality — enabling near-real-time generation.

SANA: Efficiency Through Linear Attention

SANA (NVIDIA, ICLR 2025 Oral) demonstrated that competitive image generation does not require 10B+ parameters. At just 0.6B parameters, SANA is 20× smaller than FLUX and 100× faster in throughput. Three innovations enable this. First, linear attention replaces quadratic self-attention, reducing complexity from $O(n^2)$ to $O(n)$ for the token count $n$ (the product $n \cdot d$ scales linearly rather than quadratically). Second, Mix-FFN adds 3×3 depthwise convolutions to the MLP, injecting local spatial information that helps compensate for the weaker global modeling of linear attention. Third, a Deep Compression Autoencoder (DC-AE) compresses images with a 32× spatial factor — producing 16× fewer latent tokens than the standard 8× autoencoder. SANA generates 1024×1024 images in under one second on a 16GB laptop GPU.

GPT Image 1.5

GPT Image 1.5 (OpenAI, late 2025) replaced DALL-E 3 with a fundamentally different approach: native multimodal generation within a single neural network. Unlike DALL-E 3, which used a separate diffusion model conditioned on CLIP embeddings, GPT Image 1.5 processes text and images in the same transformer. The result is exceptional text rendering — the model understands character-level layout because it processes text as a first-class modality, not a conditioning signal.

Key Insight

The frontier models reveal a surprising divide. Scale-first models (FLUX 2 at 32B, Midjourney V8) pursue maximum quality through raw parameter count and data curation. Efficiency-first models (SANA at 0.6B) pursue deployment accessibility through architectural innovation — linear attention, aggressive compression, compact encoders. Both achieve results that would have been considered state-of-the-art 18 months ago. The lesson: in mature architectures, both scaling and efficiency deliver diminishing-returns-free improvements.

Controllability and Editing

Raw text-to-image generation gives the user only a text prompt as control. Creative applications demand finer control: preserving the structure of a reference image, maintaining the style of a reference, or editing specific regions. Three techniques address this.

ControlNet

ControlNet injects spatial conditioning — depth maps, edge maps, pose skeletons, segmentation masks — into the diffusion model through a parallel branch. The conditioning encoder is initialized as a copy of the main model's encoder, with zero-initialized convolutions at each connection point. This zero initialization ensures the ControlNet has no effect at the start of training, preserving the base model's quality, and gradually learns to incorporate the spatial signal. The result: structure-preserving generation that follows the spatial layout of a reference while generating new content, textures, and styles.

IP-Adapter: Image Prompting

IP-Adapter enables image-based conditioning through decoupled cross-attention. A CLIP image encoder extracts features from a reference image. These features are injected into the diffusion model through a separate cross-attention pathway — distinct from the text cross-attention — so that text and image conditioning do not interfere with each other. The decoupled design means the model can simultaneously follow a text prompt ("a cat sitting on a couch") and adopt the style/content of a reference image (a specific art style, color palette, or object appearance).

The following code demonstrates a simplified ControlNet zero-initialization pattern.


import torch
import torch.nn as nn

class ControlNetBlock(nn.Module):
    """ControlNet conditioning with zero-initialized connection."""
    def __init__(self, d_model=1152):
        super().__init__()
        # Conditioning encoder (e.g., for depth maps)
        self.cond_encoder = nn.Sequential(
            nn.Conv2d(1, d_model // 4, 3, padding=1), nn.SiLU(),
            nn.Conv2d(d_model // 4, d_model // 2, 3, stride=2, padding=1), nn.SiLU(),
            nn.Conv2d(d_model // 2, d_model, 3, stride=2, padding=1),  # downsample to latent res
        )
        # Zero-initialized projection — starts with no effect on base model
        self.zero_conv = nn.Conv2d(d_model, d_model, 1)
        nn.init.zeros_(self.zero_conv.weight)
        nn.init.zeros_(self.zero_conv.bias)

    def forward(self, base_features, control_image):
        # base_features: (B, d_model, H, W) — from base diffusion model
        # control_image: (B, 1, 4H, 4W) — e.g., depth map at higher resolution
        cond = self.cond_encoder(control_image)    # (B, d_model, H, W)
        cond = self.zero_conv(cond)                # (B, d_model, H, W) — starts at zero
        return base_features + cond                # additive conditioning

InstructPix2Pix: Instruction-Based Editing

InstructPix2Pix fine-tunes a diffusion model to follow editing instructions in natural language. Given an input image and an instruction ("make it sunset," "add snow to the mountains"), the model generates an edited version. The training data is generated synthetically: GPT-4 produces editing instructions, and a pretrained image model applies the edits. The model learns to apply localized changes while preserving the rest of the image — a capability that standard text-to-image models, which generate from scratch each time, cannot provide.

Every controllability technique in this section — ControlNet, IP-Adapter, instruction-based editing — extends directly to video in the next two chapters, where temporal consistency adds another dimension of challenge.
