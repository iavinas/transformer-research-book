Chapter 23c

Training Video Generation Transformers

Generating a single 512×512 image requires denoising 16,384 latent tokens. Generating a 10-second, 24-fps video at the same resolution means denoising 3.9 million tokens — a 240× increase that makes naive approaches computationally impossible. Every technique in video generation training exists to tame this cost: 3D VAEs compress the temporal dimension alongside the spatial, pyramidal flow matching processes distant frames at lower resolution, progressive pipelines break the problem into manageable stages, and mixed image-video training prevents the model from forgetting how to produce sharp individual frames. This chapter walks through each technique with complete PyTorch code, from training a 3D VAE that compresses video 512× to running the full Open-Sora 2.0 progressive pipeline that produces commercial-quality video for $200K in total training cost.

3D VAE Training for Video Compression

Image diffusion uses a 2D VAE that compresses spatial dimensions by 8×. Video diffusion needs temporal compression too — processing every frame independently would be prohibitively expensive and would miss temporal redundancy (adjacent frames are nearly identical). A 3D VAE compresses along all three dimensions: spatial height and width by 8×, and the temporal axis by 4-8×. A 10-second, 24-fps video at 256×256 (240 frames × 256 × 256 × 3 ≈ 47M values) becomes a latent of shape 30 × 32 × 32 × 4 (≈ 123K values) — a 380× compression.

From 2D VAE to 3D: Temporal Compression

The standard approach initializes a 3D VAE from a pretrained 2D image VAE: the spatial convolution weights are copied directly, and new temporal convolution weights are initialized to identity (pass-through). This means at initialization, the 3D VAE behaves identically to a 2D VAE applied frame-by-frame — temporal compression is learned from scratch while spatial compression starts from a strong prior.

The temporal compression is introduced in the last two downsampling blocks of the encoder (and first two upsampling blocks of the decoder), using causal 3D convolutions — convolutions that only look at the current and past frames, never future frames. This causality is critical for autoregressive video generation where frames are produced sequentially.

Group Causal Convolution (CVPR 2025)

Standard causal convolution pads the temporal dimension asymmetrically (all padding on the past side), which causes the first frame to see a different receptive field than later frames. This creates visible quality inconsistency: the first frame is sharp, subsequent frames are slightly blurry. Group causal convolution fixes this by dividing the temporal axis into groups, applying standard (non-causal) convolution within each group, and causal padding only between groups. The result: consistent quality across all frames, with the causal property preserved at the group boundary level.

Hands-On: 3D VAE Encoder-Decoder

The following code implements a simplified 3D VAE with temporal compression, demonstrating the key architectural patterns.


import torch
import torch.nn as nn
import torch.nn.functional as F

class CausalConv3d(nn.Module):
    """3D convolution that is causal in the temporal dimension."""
    def __init__(self, in_ch, out_ch, kernel_size=3, stride=(1,1,1)):
        super().__init__()
        # Causal: pad only the past in temporal dim, symmetric in spatial
        t_k = kernel_size if isinstance(kernel_size, int) else kernel_size[0]
        self.t_pad = t_k - 1                                  # full causal padding
        self.conv = nn.Conv3d(in_ch, out_ch, kernel_size,
                              stride=stride, padding=(0, 1, 1))  # spatial pad only

    def forward(self, x):
        # x: (B, C, T, H, W)
        x = F.pad(x, (0, 0, 0, 0, self.t_pad, 0))            # pad past frames
        return self.conv(x)

class VideoEncoder3D(nn.Module):
    """3D VAE encoder: spatial ↓8×, temporal ↓4×."""
    def __init__(self, in_ch=3, latent_ch=4, base_ch=64):
        super().__init__()
        self.spatial_down = nn.Sequential(
            # Spatial-only downsampling (stride in H,W only)
            nn.Conv3d(in_ch, base_ch, 3, stride=(1,2,2), padding=1),   # H/2, W/2
            nn.SiLU(),
            nn.Conv3d(base_ch, base_ch*2, 3, stride=(1,2,2), padding=1), # H/4, W/4
            nn.SiLU(),
            nn.Conv3d(base_ch*2, base_ch*4, 3, stride=(1,2,2), padding=1), # H/8, W/8
            nn.SiLU(),
        )
        self.temporal_down = nn.Sequential(
            # Temporal downsampling with causal convolutions
            CausalConv3d(base_ch*4, base_ch*4, kernel_size=3, stride=(2,1,1)), # T/2
            nn.SiLU(),
            CausalConv3d(base_ch*4, base_ch*4, kernel_size=3, stride=(2,1,1)), # T/4
            nn.SiLU(),
        )
        self.to_latent = nn.Conv3d(base_ch*4, latent_ch * 2, 1)  # mu + logvar

    def forward(self, video):
        """video: (B, 3, T, H, W) → latent: (B, 4, T/4, H/8, W/8)."""
        h = self.spatial_down(video)                           # (B, 256, T, H/8, W/8)
        h = self.temporal_down(h)                              # (B, 256, T/4, H/8, W/8)
        mu, logvar = self.to_latent(h).chunk(2, dim=1)        # each (B, 4, T/4, H/8, W/8)
        z = mu + (0.5 * logvar).exp() * torch.randn_like(mu)  # reparameterize: μ + σ ⊙ ε
        return z, mu, logvar

class VideoDecoder3D(nn.Module):
    """3D VAE decoder: spatial ↑8×, temporal ↑4×."""
    def __init__(self, latent_ch=4, out_ch=3, base_ch=64):
        super().__init__()
        self.from_latent = nn.Conv3d(latent_ch, base_ch*4, 1)
        self.temporal_up = nn.Sequential(
            nn.ConvTranspose3d(base_ch*4, base_ch*4, (4,1,1), stride=(2,1,1),
                               padding=(1,0,0)),              # T×2
            nn.SiLU(),
            nn.ConvTranspose3d(base_ch*4, base_ch*4, (4,1,1), stride=(2,1,1),
                               padding=(1,0,0)),              # T×4
            nn.SiLU(),
        )
        self.spatial_up = nn.Sequential(
            nn.ConvTranspose3d(base_ch*4, base_ch*2, (1,4,4), stride=(1,2,2),
                               padding=(0,1,1)),              # H×2, W×2
            nn.SiLU(),
            nn.ConvTranspose3d(base_ch*2, base_ch, (1,4,4), stride=(1,2,2),
                               padding=(0,1,1)),              # H×4, W×4
            nn.SiLU(),
            nn.ConvTranspose3d(base_ch, out_ch, (1,4,4), stride=(1,2,2),
                               padding=(0,1,1)),              # H×8, W×8
            nn.Tanh(),
        )

    def forward(self, z):
        """z: (B, 4, T/4, H/8, W/8) → video: (B, 3, T, H, W)."""
        h = self.from_latent(z)
        h = self.temporal_up(h)
        return self.spatial_up(h)

# --- Demo: encode and decode a video clip ---

decoder = VideoDecoder3D().cuda()

video = torch.randn(1, 3, 16, 256, 256).cuda()                # 16-frame clip

Video Diffusion Transformer Training

Once the 3D VAE compresses video into a manageable latent, the diffusion transformer learns to denoise it. The core training loop is identical to image DiT (Chapter 23b) — sample $z_0$, add noise to get $z_t$, predict velocity — but the transformer architecture must handle the temporal dimension. Two major approaches have emerged: factorized spatial-temporal attention (cheaper, used by Open-Sora) and full 3D attention (better quality, used by CogVideoX).

CogVideoX: Expert Transformer with 3D Full Attention

CogVideoX processes text and video tokens through full 3D attention — every token at every position in every frame attends to every other. To handle the distributional difference between text and video tokens (text features live in a different numerical range), CogVideoX uses Expert Adaptive LayerNorm: separate LayerNorm modules for text and video tokens, each parameterized by the diffusion timestep $t$. The text-stream normalization parameters $(\gamma_t^{\text{text}}, \beta_t^{\text{text}})$ and video-stream parameters $(\gamma_t^{\text{vid}}, \beta_t^{\text{vid}})$ are predicted from separate MLPs conditioned on $t$.

3D RoPE extends rotary position embeddings to three dimensions. Each token's position $(t, h, w)$ is encoded by rotating its embedding vector in frequency bands assigned to each axis. Temporal frequencies use lower bands (capturing slow motion), spatial frequencies use higher bands (capturing fine texture). This allows the model to generalize to video lengths and resolutions unseen during training.

Hands-On: Video Diffusion Training Step

The following code implements a complete video diffusion training step with factorized spatial-temporal attention.


import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SpatialTemporalBlock(nn.Module):
    """Factorized spatial-temporal transformer block for video."""
    def __init__(self, dim=768, heads=12):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.spatial_attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim)
        self.temporal_attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.norm3 = nn.LayerNorm(dim)
        self.mlp = nn.Sequential(
            nn.Linear(dim, dim * 4), nn.GELU(), nn.Linear(dim * 4, dim))
        # AdaLN for timestep conditioning
        self.adaLN = nn.Sequential(nn.SiLU(), nn.Linear(dim, 6 * dim))

    def forward(self, x, cond, T, H, W):
        """
        x: (B, T*H*W, D) flattened video tokens
        cond: (B, D) timestep + text conditioning
        T, H, W: temporal and spatial dimensions
        """
        B, N, D = x.shape
        shift1, scale1, gate1, shift2, scale2, gate2 = \
            self.adaLN(cond).chunk(6, dim=-1)

        # --- Spatial attention: attend within each frame ---
        h = self.norm1(x) * (1 + scale1.unsqueeze(1)) + shift1.unsqueeze(1)
        h = h.reshape(B * T, H * W, D)                        # merge batch and time
        h, _ = self.spatial_attn(h, h, h)                      # (B*T, H*W, D)
        h = h.reshape(B, T * H * W, D)
        x = x + gate1.unsqueeze(1) * h

        # --- Temporal attention: attend across frames at same position ---
        h = self.norm2(x)
        h = h.reshape(B, T, H * W, D).permute(0, 2, 1, 3)    # (B, H*W, T, D)
        h = h.reshape(B * H * W, T, D)                        # merge batch and spatial
        h, _ = self.temporal_attn(h, h, h)                     # (B*H*W, T, D)
        h = h.reshape(B, H * W, T, D).permute(0, 2, 1, 3)    # (B, T, H*W, D)
        h = h.reshape(B, T * H * W, D)
        x = x + h

        # --- FFN ---
        h = self.norm3(x) * (1 + scale2.unsqueeze(1)) + shift2.unsqueeze(1)
        x = x + gate2.unsqueeze(1) * self.mlp(h)
        return x

class VideoDiT(nn.Module):
    """Video Diffusion Transformer with factorized attention."""
    def __init__(self, latent_ch=4, patch_size=2, dim=768, depth=12, heads=12):
        super().__init__()
        self.patch_size = patch_size
        # 3D patchify: (B, C, T, H, W) → (B, T*(H/p)*(W/p), D)
        self.proj = nn.Conv3d(latent_ch, dim, (1, patch_size, patch_size),
                              stride=(1, patch_size, patch_size))
        self.time_embed = nn.Sequential(
            nn.Linear(256, dim), nn.SiLU(), nn.Linear(dim, dim))
        self.blocks = nn.ModuleList([
            SpatialTemporalBlock(dim, heads) for _ in range(depth)])
        self.final_norm = nn.LayerNorm(dim)
        self.head = nn.Linear(dim, patch_size**2 * latent_ch)

    def timestep_embedding(self, t, dim=256):
        half = dim // 2
        freqs = torch.exp(-math.log(10000) * torch.arange(half, device=t.device) / half)
        args = t[:, None] * freqs[None]
        return torch.cat([args.cos(), args.sin()], dim=-1)

    def forward(self, z_t, t, text_cond):
        """z_t: (B, 4, T, H, W), t: (B,), text_cond: (B, D)."""
        B, C, T, H, W = z_t.shape
        ps = self.patch_size
        pH, pW = H // ps, W // ps                             # patched spatial dims

        x = self.proj(z_t)                                     # (B, D, T, pH, pW)
        x = x.flatten(2).transpose(1, 2)                      # (B, T*pH*pW, D)

        cond = self.time_embed(self.timestep_embedding(t)) + text_cond

        for blk in self.blocks:
            x = blk(x, cond, T, pH, pW)

        x = self.head(self.final_norm(x))                     # (B, T*pH*pW, ps²*C)
        # Unpatchify back to (B, C, T, H, W)
        x = x.reshape(B, T, pH, pW, ps, ps, C)
        x = x.permute(0, 6, 1, 2, 4, 3, 5).reshape(B, C, T, H, W)
        return x                                               # predicted velocity

# --- Training step ---
def video_dit_training_step(model, vae_encoder, video, text_cond, optimizer):
    """Single training step for video DiT with rectified flow."""
    B = video.size(0)

    # Encode video to latent
    with torch.no_grad():
        z_0, _, _ = vae_encoder(video)                         # (B, 4, T/4, H/8, W/8)

    # Sample timestep and noise
    t = torch.rand(B, device=z_0.device)
    eps = torch.randn_like(z_0)
    t_exp = t.view(B, 1, 1, 1, 1)                             # broadcast to 5D
    z_t = (1 - t_exp) * z_0 + t_exp * eps                     # rectified flow

    # Predict velocity
    v_pred = model(z_t, t, text_cond)                          # (B, 4, T', H', W')
    v_target = eps - z_0                                       # velocity target

    loss = F.mse_loss(v_pred, v_target)
    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    return loss.item()

Mixed Image-Video Training

A model trained exclusively on video data forgets how to generate high-quality still images. The frames become temporally coherent but individually blurry — the model optimizes for consistency at the expense of sharpness. Mixed image-video training solves this by including both image and video data in each training batch.

Domain-Specific Normalization

Images and videos have different activation statistics — a video batch has temporal correlations that make its feature distributions narrower than an image batch. Using shared batch normalization produces suboptimal statistics for both. The solution: maintain separate normalization layers for images and videos while sharing all other parameters. Open-Sora 2.0 implements this as separate LayerNorm affine parameters $(\gamma_{\text{img}}, \beta_{\text{img}})$ and $(\gamma_{\text{vid}}, \beta_{\text{vid}})$ that are selected based on the input type. CogVideoX uses Expert Adaptive LayerNorm for the same purpose.

Handling Format Differences

Images are treated as single-frame videos: an image $x \in \mathbb{R}^{3 \times H \times W}$ is zero-padded to match the video temporal dimension $\hat{x} \in \mathbb{R}^{3 \times T \times H \times W}$ where frames $2, 3, \ldots, T$ are zeros. The 3D VAE encodes this as if it were a video with a static first frame. During mixed training, Open-Sora 2.0 randomizes video lengths (up to 34 frames) with zero-padding, improving robustness to varying durations at inference time.

The following code demonstrates how to set up a mixed image-video data pipeline.


import torch
from datasets import load_dataset
from torchvision import transforms

# --- Mixed image-video collate function ---
def mixed_collate(image_batch, video_batch, max_frames=17):
    """Combine image and video samples into a unified batch.

    Images are zero-padded to max_frames. Returns batch with
    a domain flag for normalization routing.
    """
    batch_videos = []
    domain_flags = []                                          # 0=image, 1=video

    # Process images: treat as single-frame "videos"
    img_transform = transforms.Compose([
        transforms.Resize(256), transforms.CenterCrop(256),
        transforms.ToTensor(), transforms.Normalize([0.5]*3, [0.5]*3),
    ])
    for item in image_batch:
        img = img_transform(item["image"].convert("RGB"))      # (3, 256, 256)
        # Zero-pad to (3, max_frames, 256, 256)
        padded = torch.zeros(3, max_frames, 256, 256)
        padded[:, 0] = img                                     # first frame = image
        batch_videos.append(padded)
        domain_flags.append(0)                                 # image domain

    # Process videos: already temporal
    for frames in video_batch:
        T = min(len(frames), max_frames)
        video = torch.zeros(3, max_frames, 256, 256)
        for i in range(T):
            video[:, i] = img_transform(frames[i])
        batch_videos.append(video)
        domain_flags.append(1)                                 # video domain

    return (torch.stack(batch_videos),                         # (B, 3, T, 256, 256)
            torch.tensor(domain_flags))                        # (B,)

class DomainNorm(nn.Module):
    """Domain-specific normalization: separate stats for images vs videos."""
    def __init__(self, dim):
        super().__init__()
        self.img_norm = nn.LayerNorm(dim)
        self.vid_norm = nn.LayerNorm(dim)

    def forward(self, x, domain_flag):
        """x: (B, N, D), domain_flag: (B,) with 0=image, 1=video."""
        out = torch.zeros_like(x)
        img_mask = domain_flag == 0
        vid_mask = domain_flag == 1
        if img_mask.any():
            out[img_mask] = self.img_norm(x[img_mask])
        if vid_mask.any():
            out[vid_mask] = self.vid_norm(x[vid_mask])
        return out

Progressive Training Pipeline

Training a video model end-to-end at full resolution and duration is prohibitively expensive. The solution is progressive training: start simple, scale up gradually, and transfer checkpoints between stages. Open-Sora 2.0 demonstrated that this approach produces commercial-quality video generation for a total training cost of $200K — 5-10× cheaper than comparable models.

Stage Transitions: What to Keep, What to Reinitialize

When transitioning from Stage 1 (image) to Stage 2 (video), all spatial layers transfer directly — the model already knows how to generate spatial content. The temporal attention layers (new in Stage 2) are initialized to identity: the temporal attention output projection's weights are set to zero, so the model initially behaves as a per-frame image generator. The domain-specific normalization parameters for video are initialized from the image normalization parameters. Learning rate is typically reduced by 2-5× for transferred layers and kept at the original rate for new layers.

From Stage 2 to Stage 3, all parameters transfer. The main changes are in the data pipeline: higher resolution images/videos and longer clips. Positional embeddings are interpolated (bicubic for spatial, linear for temporal) to handle the resolution increase. Pyramidal flow matching (next section) is introduced in Stage 3 to handle the dramatically increased token count.

Pyramidal Flow Matching for Efficient Training

A 10-second, 241-frame video at 768p resolution contains approximately 119,000 tokens in latent space. Full attention over this sequence costs $O(119000^2) \approx 14$ billion operations per layer — impractical even on modern hardware. Pyramidal flow matching (ICLR 2025) reduces this by processing the video at multiple resolutions simultaneously: recent frames at full resolution, and progressively older frames at lower resolution.

Training the Pyramid

During training, the model processes all pyramid levels in a single forward pass. The denoising loss is computed at each level, but with different noise schedules: lower-resolution levels use noisier inputs (higher $t$ values), reflecting that coarse structure should be resolved first. The training objective is a weighted sum across levels:

$$\mathcal{L}_{\text{pyramid}} = \sum_{l=1}^{L} w_l \, \mathbb{E}_{t_l}\!\left[\|v_\theta^{(l)}(z_{t_l}^{(l)}) - v^{(l)}\|^2\right]$$

where $l$ indexes the pyramid level, $w_l$ is the level weight (typically higher for the full-resolution level), $t_l$ is the noise level for level $l$ (lower resolution = noisier), and $v_\theta^{(l)}$ is the velocity prediction at level $l$. The element-wise norm $\|\cdot\|^2$ is computed independently at each spatial resolution (element-wise squared difference summed over all values in the level's tensor).

Hands-On: Pyramidal Flow Matching Training Step

The following code implements a simplified pyramidal flow matching training step for video generation.


import torch
import torch.nn.functional as F

def build_pyramid(z_0, levels=3):
    """Build a temporal-spatial pyramid from a full-resolution latent.
    Args:
        z_0: (B, C, T, H, W) clean video latent at full resolution
        levels: number of pyramid levels (3 = full, half, quarter)
    Returns:
        list of (z_level, (T_l, H_l, W_l)) from coarsest to finest
    """
    pyramid = []
    z = z_0
    for l in range(levels):
        if l > 0:
            # Downsample spatial dims by 2× using trilinear interpolation
            z = F.interpolate(z, scale_factor=(1, 0.5, 0.5),
                              mode='trilinear', align_corners=False)
        pyramid.append(z)
    return list(reversed(pyramid))                             # coarsest first

def pyramid_training_step(model, z_0, text_cond, optimizer, levels=3):
    """One training step with pyramidal flow matching.

    Each pyramid level uses a different noise schedule:
    coarse levels see more noise, fine levels see less.
    """
    B = z_0.size(0)
    pyramid = build_pyramid(z_0, levels)                       # [coarse, ..., fine]

    total_loss = 0
    level_weights = [0.2, 0.3, 0.5]                           # higher weight for fine

    for l, (z_l, w_l) in enumerate(zip(pyramid, level_weights)):
        # Noise schedule: coarser levels are noisier
        t_min = 0.3 * (levels - 1 - l) / (levels - 1)        # coarse: t_min=0.3, fine: 0
        t_max = 0.7 + 0.3 * (levels - 1 - l) / (levels - 1)  # coarse: t_max=1.0, fine: 0.7
        t = torch.rand(B, device=z_l.device) * (t_max - t_min) + t_min

        eps = torch.randn_like(z_l)
        t_exp = t.view(B, 1, 1, 1, 1)
        z_t = (1 - t_exp) * z_l + t_exp * eps                 # noisy latent at level l

        v_pred = model(z_t, t, text_cond, level=l)            # model receives level info
        v_target = eps - z_l
        total_loss += w_l * F.mse_loss(v_pred, v_target)

    optimizer.zero_grad()
    total_loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    return total_loss.item()

Data Curation and Filtering for Video Training

Video training data quality matters more than quantity. Open-Sora 2.0 trained on a curated dataset that is orders of magnitude smaller than the raw web video corpus it was drawn from — but dramatically cleaner. The data pipeline has four stages.

Filtering Pipeline

Aesthetic filtering removes visually unappealing clips: low resolution, heavy compression artifacts, text overlays, watermarks. A pretrained aesthetic scoring model (typically a CLIP-based classifier fine-tuned on human preference data) assigns each clip a quality score, and clips below a threshold are removed. Motion filtering removes static clips (slideshows, still images with music) by computing optical flow magnitude across frames — clips with average flow below a threshold are discarded. At the other extreme, clips with erratic camera motion (shaky handheld footage) are also filtered.

Caption generation uses a VLM (such as LLaVA or CogVLM) to generate dense textual descriptions for each video clip. Frame-level captions are generated at regular intervals and then consolidated into a single video-level caption that describes the scene, actions, camera movement, and temporal progression. High-quality captions are critical for text-video alignment — the model can only learn the mapping from text to video if the text accurately describes the visual content.

Deduplication removes near-duplicate clips (common in web video) using embedding-based similarity: each clip is embedded with a video encoder, and clips with cosine similarity above 0.95 are deduplicated. This prevents the model from memorizing specific clips and improves diversity.

Key Insight

The data pipeline for video generation training typically reduces the raw dataset by 90-95%. Out of millions of web video clips, only the top 5-10% survive aesthetic, motion, caption quality, and deduplication filtering. This aggressive curation is why Open-Sora 2.0 achieves commercial quality at $200K — the model trains on fewer but much higher quality examples. The saying "garbage in, garbage out" is especially true for video: a single batch of low-quality clips can destabilize training for thousands of subsequent steps.

Video Generation Training Pitfalls

The three most common failures in video generation training: (1) Temporal flicker — if the 3D VAE's temporal compression is too aggressive or undertrained, decoded videos flicker between frames. Always verify VAE temporal reconstruction quality before training the DiT. (2) Mode collapse to static video — without motion filtering in the data pipeline and without mixed-duration training, the model learns to generate the easiest output: a single repeated frame. Monitor motion magnitude in generated videos during training. (3) Text-video misalignment — poor-quality captions cause the model to ignore the text prompt. Invest in caption quality: VLM-generated captions are better than metadata-derived ones, and caption filtering (removing low-confidence captions) is critical.
