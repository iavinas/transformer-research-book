Chapter 23b

Training Image Generation Transformers

Diffusion Transformers have replaced U-Nets as the dominant backbone for image generation, powering FLUX, Stable Diffusion 3.5, and SANA. But training a DiT is nothing like training a classifier — the loss landscape is different, the conditioning mechanisms are different, and the path from a trained model to a fast inference model requires an entirely separate distillation stage. This chapter walks through every step of the image generation training pipeline with complete PyTorch code: encoding images into latent space, training a DiT with rectified flow and classifier-free guidance, distilling a 50-step model into a 1-step generator, and training conditional adapters like ControlNet. Each recipe uses real data from HuggingFace datasets and produces a model you can sample from.

Latent Diffusion Fundamentals for Transformers

Generating a 512×512 image directly in pixel space requires a transformer to process 262,144 pixels — far too many tokens for self-attention. Latent diffusion solves this by compressing images into a much smaller latent space using a pretrained Variational Autoencoder (VAE), running the diffusion process in that compressed space, and decoding back to pixels only at the end. The compression ratio is dramatic: the standard SD VAE with a spatial downsampling factor $f = 8$ and 4 latent channels compresses a 512×512×3 image (786,432 values) into a 64×64×4 latent (16,384 values) — a 48× reduction.

The SD VAE: Encoding and Decoding

The standard Stable Diffusion VAE is a convolutional autoencoder trained with a combination of pixel reconstruction loss, perceptual loss (from a pretrained VGG), adversarial loss (patch discriminator), and KL regularization on the latent distribution. The encoder maps an image $x \in \mathbb{R}^{H \times W \times 3}$ to a distribution $q(z|x) = \mathcal{N}(\mu(x), \sigma^2(x))$, where $\mu$ and $\sigma$ are spatial feature maps of shape $\frac{H}{f} \times \frac{W}{f} \times C_z$. A latent sample is drawn as $z = \mu + \sigma \odot \epsilon$ (element-wise product between the learned standard deviation $\sigma \in \mathbb{R}^{h \times w \times C_z}$ and random noise $\epsilon \sim \mathcal{N}(0, I)$ of the same shape, known as the reparameterization trick). In practice, the latent is rescaled by a constant factor (0.18215 for the SD 1.x/2.x VAE) so that the initial latent distribution has approximately unit variance.

The following code demonstrates encoding images to latent space and decoding back, using ImageNet images loaded from HuggingFace datasets.


import torch
import torch.nn as nn
from torchvision import transforms
from datasets import load_dataset

# --- Simplified VAE Encoder / Decoder ---
class SimpleVAEEncoder(nn.Module):
    """Downsample 3-channel image to 4-channel latent (f=8)."""
    def __init__(self, in_ch=3, latent_ch=4):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Conv2d(in_ch, 64, 3, stride=2, padding=1),    # H/2
            nn.SiLU(),
            nn.Conv2d(64, 128, 3, stride=2, padding=1),      # H/4
            nn.SiLU(),
            nn.Conv2d(128, 256, 3, stride=2, padding=1),     # H/8
            nn.SiLU(),
            nn.Conv2d(256, latent_ch * 2, 3, padding=1),     # mu + logvar
        )

    def forward(self, x):
        h = self.encoder(x)                                   # (B, 8, H/8, W/8)
        mu, logvar = h.chunk(2, dim=1)                        # each (B, 4, H/8, W/8)
        std = (0.5 * logvar).exp()
        z = mu + std * torch.randn_like(std)                  # reparameterization: μ + σ ⊙ ε
        return z, mu, logvar                                  # z: (B, 4, H/8, W/8)

class SimpleVAEDecoder(nn.Module):
    """Upsample 4-channel latent back to 3-channel image."""
    def __init__(self, latent_ch=4, out_ch=3):
        super().__init__()
        self.decoder = nn.Sequential(
            nn.ConvTranspose2d(latent_ch, 256, 4, stride=2, padding=1),  # H/4
            nn.SiLU(),
            nn.ConvTranspose2d(256, 128, 4, stride=2, padding=1),        # H/2
            nn.SiLU(),
            nn.ConvTranspose2d(128, 64, 4, stride=2, padding=1),         # H
            nn.SiLU(),
            nn.Conv2d(64, out_ch, 3, padding=1),
            nn.Tanh(),
        )

    def forward(self, z):
        return self.decoder(z)                                # (B, 3, H, W)

# --- Usage with HuggingFace datasets ---
transform = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(256),
    transforms.ToTensor(), transforms.Normalize([0.5]*3, [0.5]*3),
])
ds = load_dataset("imagenet-1k", split="train", streaming=True)

encoder = SimpleVAEEncoder().cuda()
decoder = SimpleVAEDecoder().cuda()

for item in ds:
    img = transform(item["image"].convert("RGB")).unsqueeze(0).cuda()  # (1, 3, 256, 256)
    z, mu, logvar = encoder(img)                              # z: (1, 4, 32, 32)
    recon = decoder(z)                                        # (1, 3, 256, 256)
    # Loss: reconstruction + KL divergence
    recon_loss = nn.functional.mse_loss(recon, img)
    kl_loss = -0.5 * (1 + logvar - mu.pow(2) - logvar.exp()).sum() / img.size(0)
    loss = recon_loss + 1e-6 * kl_loss                        # small KL weight
    break  # demo: single step

DiT and MMDiT Training

Training a Diffusion Transformer follows the standard diffusion recipe: sample a clean data point $z_0$ from the VAE latent space, add noise at a random timestep $t$ to get $z_t$, predict the noise (or velocity), and minimize the prediction error. But the conditioning mechanism — how the model learns to use timestep, class label, or text information — is where DiT diverges from U-Net diffusion.

Rectified Flow: Linear Interpolation Between Data and Noise

Rectified flow (used by SD3, FLUX, and SANA) defines a straight-line path between data and noise. At timestep $t \in [0, 1]$, the noisy latent is:

$$z_t = (1 - t) z_0 + t \epsilon$$

where $z_0$ is the clean latent and $\epsilon \sim \mathcal{N}(0, I)$ is pure noise. The velocity at any point along this path is constant: $v = \epsilon - z_0$ (element-wise subtraction, both tensors in $\mathbb{R}^{C \times H \times W}$). The model learns to predict this velocity, and the loss is simply:

$$\mathcal{L}_v = \mathbb{E}_{t, z_0, \epsilon}\!\left[\|v_\theta(z_t, t, c) - (\epsilon - z_0)\|^2\right]$$

where $c$ is the conditioning signal (class label or text embedding). The beauty of rectified flow is its simplicity: straight paths mean fewer integration steps at inference, and the velocity target $\epsilon - z_0$ does not depend on any noise schedule coefficients.

AdaLN-Zero Conditioning

DiT conditions on timestep and class/text through Adaptive Layer Normalization with Zero initialization (AdaLN-Zero). The timestep $t$ and class embedding $c$ are combined, projected through an MLP to produce per-layer modulation parameters $(\gamma_1, \beta_1, \alpha_1, \gamma_2, \beta_2, \alpha_2)$. These modulate the output of each transformer sublayer:

$$h = \alpha_1 \odot \text{Attn}(\gamma_1 \odot \text{LN}(x) + \beta_1) + x$$

where $\gamma_1, \beta_1$ are shift and scale applied element-wise to the layer-normalized input (element-wise products, $\gamma_1 \odot \text{LN}(x)$, both $\in \mathbb{R}^{d}$, first annotated occurrence of $\odot$ in this chapter), and $\alpha_1$ gates the output before the residual connection. The "Zero" initialization sets all $\alpha$ parameters to zero at initialization, so the model starts as an identity function — a critical stability trick.

Classifier-Free Guidance Training

To enable classifier-free guidance (CFG) at inference, the model must learn to generate both conditionally and unconditionally. During training, the conditioning signal is randomly dropped with probability $p_{\text{drop}} = 0.1$-$0.2$ — replaced with a null embedding (zeros). At inference, the model produces both a conditioned prediction $v_c$ and an unconditioned prediction $v_\emptyset$, combined as:

$$v_{\text{guided}} = v_\emptyset + w (v_c - v_\emptyset)$$

where $w$ is the guidance scale (scalar multiplied element-wise with the velocity difference, typically $w = 4$-$7$ for image generation).

Hands-On: Complete DiT Training Loop

The following code implements a complete class-conditional DiT training loop on ImageNet, using HuggingFace datasets for data loading.


import torch
import torch.nn as nn
import torch.nn.functional as F
from torchvision import transforms
from datasets import load_dataset
import math

# --- AdaLN-Zero DiT Block ---
class DiTBlock(nn.Module):
    def __init__(self, dim=768, heads=12, mlp_ratio=4.0):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim, elementwise_affine=False)
        self.attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim, elementwise_affine=False)
        self.mlp = nn.Sequential(
            nn.Linear(dim, int(dim * mlp_ratio)),
            nn.GELU(),
            nn.Linear(int(dim * mlp_ratio), dim),
        )
        # AdaLN-Zero: 6 modulation params per block
        self.adaLN = nn.Sequential(
            nn.SiLU(), nn.Linear(dim, 6 * dim),
        )
        # Zero-initialize the output gates
        nn.init.zeros_(self.adaLN[1].weight)
        nn.init.zeros_(self.adaLN[1].bias)

    def forward(self, x, cond):
        """x: (B, N, D), cond: (B, D) timestep + class embedding."""
        shift1, scale1, gate1, shift2, scale2, gate2 = \
            self.adaLN(cond).chunk(6, dim=-1)                 # each (B, D)
        # Attention with AdaLN modulation
        h = self.norm1(x)
        h = h * (1 + scale1.unsqueeze(1)) + shift1.unsqueeze(1)  # modulate
        h, _ = self.attn(h, h, h)
        x = x + gate1.unsqueeze(1) * h                        # gated residual
        # FFN with AdaLN modulation
        h = self.norm2(x)
        h = h * (1 + scale2.unsqueeze(1)) + shift2.unsqueeze(1)
        h = self.mlp(h)
        x = x + gate2.unsqueeze(1) * h
        return x

class DiT(nn.Module):
    def __init__(self, latent_ch=4, patch_size=2, dim=768, depth=12,
                 heads=12, num_classes=1000):
        super().__init__()
        self.patch_size = patch_size
        self.proj = nn.Conv2d(latent_ch, dim, patch_size, stride=patch_size)
        self.num_patches = (32 // patch_size) ** 2             # 256 for 32×32 latent
        self.pos_embed = nn.Parameter(torch.zeros(1, self.num_patches, dim))
        # Timestep + class conditioning
        self.time_embed = nn.Sequential(
            nn.Linear(256, dim), nn.SiLU(), nn.Linear(dim, dim),
        )
        self.class_embed = nn.Embedding(num_classes + 1, dim)  # +1 for null class
        self.blocks = nn.ModuleList([DiTBlock(dim, heads) for _ in range(depth)])
        self.final_norm = nn.LayerNorm(dim, elementwise_affine=False)
        self.final_adaLN = nn.Sequential(nn.SiLU(), nn.Linear(dim, 2 * dim))
        self.head = nn.Linear(dim, patch_size**2 * latent_ch)  # predict patch pixels
        nn.init.zeros_(self.head.weight)
        nn.init.zeros_(self.head.bias)

    def timestep_embedding(self, t, dim=256):
        """Sinusoidal timestep embedding."""
        half = dim // 2
        freqs = torch.exp(-math.log(10000) * torch.arange(half, device=t.device) / half)
        args = t[:, None] * freqs[None]
        return torch.cat([args.cos(), args.sin()], dim=-1)     # (B, 256)

    def forward(self, z_t, t, class_labels):
        """z_t: (B, 4, 32, 32), t: (B,), class_labels: (B,)."""
        B = z_t.size(0)
        x = self.proj(z_t).flatten(2).transpose(1, 2)         # (B, 256, 768)
        x = x + self.pos_embed
        cond = self.time_embed(self.timestep_embedding(t)) + \
               self.class_embed(class_labels)                  # (B, 768)
        for blk in self.blocks:
            x = blk(x, cond)
        # Final layer with AdaLN
        shift, scale = self.final_adaLN(cond).chunk(2, dim=-1)
        x = self.final_norm(x) * (1 + scale.unsqueeze(1)) + shift.unsqueeze(1)
        x = self.head(x)                                      # (B, 256, patch²×4)
        # Unpatchify: (B, 256, 16) → (B, 4, 32, 32)
        ps = self.patch_size
        h = w = 32 // ps
        x = x.reshape(B, h, w, ps, ps, 4).permute(0, 5, 1, 3, 2, 4)
        return x.reshape(B, 4, 32, 32)                        # predicted velocity

# --- Training Loop ---
transform = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(256),
    transforms.ToTensor(), transforms.Normalize([0.5]*3, [0.5]*3),
])
ds = load_dataset("imagenet-1k", split="train", streaming=True)

dit = DiT(depth=12, dim=768, heads=12).cuda()
vae_encoder = SimpleVAEEncoder().cuda().eval()                 # pretrained, frozen
optimizer = torch.optim.AdamW(dit.parameters(), lr=1e-4, weight_decay=0.0)
cfg_drop = 0.1                                                # CFG dropout probability
null_class = 1000                                              # null class index

for step, item in enumerate(ds):
    img = transform(item["image"].convert("RGB")).unsqueeze(0).cuda()
    label = torch.tensor([item["label"]], device="cuda")

    # Encode to latent
    with torch.no_grad():
        z_0, _, _ = vae_encoder(img)                           # (1, 4, 32, 32)

    # Random timestep and noise
    t = torch.rand(1, device="cuda")                           # (1,)
    eps = torch.randn_like(z_0)                                # (1, 4, 32, 32)
    z_t = (1 - t.view(1, 1, 1, 1)) * z_0 + t.view(1, 1, 1, 1) * eps

    # CFG: randomly drop class conditioning
    if torch.rand(1).item() < cfg_drop:
        label = torch.tensor([null_class], device="cuda")

    # Forward + loss
    v_pred = dit(z_t, t, label)                                # (1, 4, 32, 32)
    v_target = eps - z_0                                       # velocity target
    loss = F.mse_loss(v_pred, v_target)

    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(dit.parameters(), 1.0)
    optimizer.step()

    if step % 1000 == 0:
        print(f"Step {step}, Loss: {loss.item():.4f}")

Training Text-to-Image Models

MMDiT: Dual-Stream Text-Image Processing

Stable Diffusion 3 and FLUX use the Multi-Modal Diffusion Transformer (MMDiT), which processes text and image tokens through parallel transformer streams that interact via joint attention. Text tokens from T5-XXL and CLIP are concatenated and processed alongside the patchified latent tokens. In each MMDiT block, text and image tokens attend to each other through shared attention — the query, key, and value projections operate on the concatenated sequence $[x_{\text{text}}; x_{\text{image}}]$ (matrix multiply, $W_Q [x_{\text{text}}; x_{\text{image}}] \in \mathbb{R}^{(n_t + n_i) \times d_k}$ where $n_t$ and $n_i$ are the text and image token counts), but separate MLPs process each modality afterwards.

FLUX (2024) introduced a hybrid architecture: the first half of the network uses double-stream MMDiT blocks (separate attention for text and image with cross-attention), while the second half uses single-stream blocks (text and image tokens are concatenated and processed jointly). This captures both modality-specific processing and deep cross-modal fusion.

Noise Schedule Design

The distribution of training timesteps $t$ matters significantly. Uniform sampling $t \sim U(0, 1)$ oversamples very noisy and very clean timesteps, which are easy to denoise. SD3 uses logit-normal sampling: $t = \sigma(\text{logit}(u))$ where $u \sim \mathcal{N}(0, 1)$, which concentrates samples around $t = 0.5$ — the perceptually relevant regime where the model must distinguish signal from noise. This shift in sampling distribution produces visibly better image quality at the same training cost.

Hands-On: Text-Conditioned DiT Training Step

The following code shows how to adapt the class-conditional DiT to text conditioning with dual text encoders.


import torch
import torch.nn as nn
import math

class TextConditionedDiT(nn.Module):
    """DiT with dual text encoder conditioning (T5 + CLIP)."""
    def __init__(self, dim=768, depth=12, heads=12, t5_dim=4096, clip_dim=768):
        super().__init__()
        # Project text embeddings to model dimension
        self.t5_proj = nn.Linear(t5_dim, dim)                  # T5-XXL: 4096 → 768
        self.clip_proj = nn.Linear(clip_dim, dim)              # CLIP: 768 → 768
        self.time_embed = nn.Sequential(
            nn.Linear(256, dim), nn.SiLU(), nn.Linear(dim, dim))
        # Cross-attention for text tokens
        self.cross_attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.blocks = nn.ModuleList([DiTBlock(dim, heads) for _ in range(depth)])
        # ... (same as DiT above for patch embed, final layer, etc.)

    def forward(self, z_t, t, t5_tokens, clip_pooled):
        """
        z_t: (B, 4, H, W) noisy latent
        t: (B,) timestep
        t5_tokens: (B, seq_len, 4096) T5-XXL text features
        clip_pooled: (B, 768) CLIP pooled text embedding
        """
        # Combine timestep + CLIP pooled as global conditioning
        t_emb = self.time_embed(self.timestep_embedding(t))    # (B, 768)
        c_pooled = self.clip_proj(clip_pooled)                 # (B, 768)
        global_cond = t_emb + c_pooled                         # (B, 768)

        # Project T5 tokens for cross-attention
        text_tokens = self.t5_proj(t5_tokens)                  # (B, seq_len, 768)

        # Patchify z_t → image tokens
        x = self.proj(z_t).flatten(2).transpose(1, 2)         # (B, N, 768)
        x = x + self.pos_embed

        for blk in self.blocks:
            x = blk(x, global_cond)                            # AdaLN conditioning
            # Cross-attend to text tokens
            x_cross, _ = self.cross_attn(x, text_tokens, text_tokens)
            x = x + x_cross                                   # add cross-attention

        return self.unpatchify(self.head(self.final_layer(x, global_cond)))

# Training with logit-normal timestep sampling:
def sample_logit_normal(batch_size, device, m=0.0, s=1.0):
    """Sample timesteps from logit-normal distribution."""
    u = torch.randn(batch_size, device=device) * s + m
    return torch.sigmoid(u)                                    # t ∈ (0, 1)

Consistency Distillation for Few-Step Generation

A trained diffusion model requires 20-50 denoising steps for quality results — each step is a full forward pass through a billion-parameter network. Consistency distillation trains a student model to shortcut the denoising trajectory, generating in 1-4 steps by learning to map any point on the trajectory directly to the final clean output.

SANA-Sprint: Continuous-Time Consistency + Adversarial Distillation

SANA-Sprint (ICCV 2025) achieves one-step generation by combining two distillation objectives. The continuous-time consistency model (sCM) loss enforces that the student's output is the same regardless of where on the trajectory you start — $f_\theta(z_{t_1}, t_1) \approx f_\theta(z_{t_2}, t_2)$ for any two points on the same trajectory. The latent adversarial distillation (LADD) loss adds a discriminator in latent space that distinguishes real latents from student-generated ones, providing sharper gradients than pure consistency loss alone. Together they achieve 7.59 FID on GenEval in just one step — 10× faster than FLUX-Schnell with better quality.

DMD2: Distribution Matching Without Regression

DMD2 (Distribution Matching Distillation 2) takes a different approach: instead of matching individual trajectories, it matches the overall distribution of generated images to the teacher's distribution. The key insight is that you do not need a regression loss (MSE between student and teacher outputs) — pure distribution matching via a learned discriminator is sufficient and produces sharper images. DMD2 achieves 1.28 FID on ImageNet-64 and scales to megapixel distillation of SDXL.

Hands-On: Consistency Distillation Training Step

The following code implements a simplified consistency distillation step where a student learns to predict the clean output from any noise level.


import torch
import torch.nn.functional as F

def consistency_distillation_step(student, teacher, ema_student,
                                   z_0, class_labels, optimizer):
    """One step of continuous-time consistency distillation.

    The student must map any z_t to the same clean output z_0,
    regardless of the timestep t.
    """
    B = z_0.size(0)
    eps = torch.randn_like(z_0)

    # Sample two adjacent timesteps on the same trajectory
    t1 = torch.rand(B, device=z_0.device) * 0.95 + 0.05       # t1 ∈ (0.05, 1.0)
    dt = 0.01                                                  # small step
    t2 = (t1 - dt).clamp(min=0.001)                           # t2 slightly cleaner

    # Create noisy latents at both timesteps (rectified flow)
    z_t1 = (1 - t1.view(B, 1, 1, 1)) * z_0 + t1.view(B, 1, 1, 1) * eps
    z_t2 = (1 - t2.view(B, 1, 1, 1)) * z_0 + t2.view(B, 1, 1, 1) * eps

    # Student predicts clean output from z_t1
    pred_from_t1 = student(z_t1, t1, class_labels)             # (B, 4, H, W)

    # EMA student predicts clean output from z_t2 (target, no gradient)
    with torch.no_grad():
        pred_from_t2 = ema_student(z_t2, t2, class_labels)    # (B, 4, H, W)

    # Consistency loss: both should predict the same clean output
    loss = F.mse_loss(pred_from_t1, pred_from_t2)

    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(student.parameters(), 1.0)
    optimizer.step()

    # Update EMA student
    with torch.no_grad():
        for p_s, p_e in zip(student.parameters(), ema_student.parameters()):
            p_e.data = 0.999 * p_e.data + 0.001 * p_s.data

    return loss.item()
Key Insight

Consistency distillation does not train a model from scratch — it compresses a pretrained teacher's knowledge into fewer inference steps. The teacher must already be a good generator. Think of it as lossy compression of the denoising trajectory: the student learns the teacher's shortcuts, trading a small quality drop for a 10-50× speed gain. The best results (SANA-Sprint, DMD2) combine consistency with adversarial objectives — the adversarial loss prevents the blurriness that pure MSE consistency tends to produce.

ControlNet and Adapter Training

A trained text-to-image model generates images from text prompts alone. ControlNet adds spatial conditioning — depth maps, edge maps, pose skeletons — by training a copy of the encoder blocks that processes the condition signal and injects its output into the main network via zero-initialized linear layers.

ControlNet Architecture and Training

ControlNet creates a trainable copy of the first half of the DiT blocks. The condition image (e.g., a Canny edge map) is encoded by a lightweight CNN and added to the copied blocks' inputs. The copied blocks' outputs are added to the corresponding layers of the frozen main model through zero-initialized convolutions — so at initialization, ControlNet has zero effect, and the model generates exactly as before. Training unfreezes only the copied blocks and the zero convolutions, keeping the main model frozen.


import torch
import torch.nn as nn

class ControlNet(nn.Module):
    """ControlNet: add spatial control to a frozen DiT."""
    def __init__(self, frozen_dit, condition_channels=1):
        super().__init__()
        # Lightweight condition encoder
        self.cond_encoder = nn.Sequential(
            nn.Conv2d(condition_channels, 64, 3, stride=2, padding=1),
            nn.SiLU(),
            nn.Conv2d(64, 128, 3, stride=2, padding=1),
            nn.SiLU(),
            nn.Conv2d(128, frozen_dit.blocks[0].norm1.normalized_shape[0],
                      3, stride=2, padding=1),                 # match DiT dim
        )
        # Copy first half of DiT blocks
        n_copy = len(frozen_dit.blocks) // 2
        self.ctrl_blocks = nn.ModuleList([
            DiTBlock(dim=768, heads=12) for _ in range(n_copy)
        ])
        # Zero-initialized output projections
        self.zero_convs = nn.ModuleList([
            nn.Linear(768, 768) for _ in range(n_copy)
        ])
        for zc in self.zero_convs:
            nn.init.zeros_(zc.weight)
            nn.init.zeros_(zc.bias)

    def forward(self, z_t, t, text_cond, spatial_cond):
        """Returns residuals to add to frozen DiT's intermediate features."""
        cond_feat = self.cond_encoder(spatial_cond)            # (B, D, h, w)
        cond_tokens = cond_feat.flatten(2).transpose(1, 2)    # (B, N, D)

        h = cond_tokens  # start from condition
        residuals = []
        for blk, zc in zip(self.ctrl_blocks, self.zero_convs):
            h = blk(h, text_cond)
            residuals.append(zc(h))                            # zero-init residual
        return residuals

IP-Adapter: Image Prompt Conditioning

IP-Adapter conditions generation on a reference image rather than spatial controls. It uses a pretrained CLIP image encoder to extract features from the reference image, then injects these through decoupled cross-attention: the original cross-attention attends to text tokens, and a new parallel cross-attention layer attends to image tokens. Only the new cross-attention layers are trained; everything else stays frozen. This enables "generate an image like this reference, but with this text description" — powerful for style transfer and character consistency.

Image Generation Training Pitfalls

The three most common failures: (1) VAE quality ceiling — your generated images can never be sharper than what the VAE can reconstruct. Always verify VAE reconstruction quality on your data before training the DiT. (2) CFG dropout too low — without enough unconditional training (≥10%), guidance at inference produces artifacts. (3) Constant LR instability — DiT training uses a constant learning rate (no cosine schedule), but this requires gradient clipping (max norm 1.0) and BF16 to avoid divergence.
