Chapter 04

Inside the Transformer Block

Chapter 3 gave you the blueprint of the original Transformer. Now we open the hood. Every modern Transformer — GPT-4, LLaMA, Gemini, PaLM — is built from a single repeating unit: the Transformer block. The specific ingredients inside that block have evolved significantly since 2017. Pre-norm replaced post-norm, SwiGLU replaced ReLU, RMS Norm replaced Layer Norm, and the "residual stream" interpretation reframed how we think about information flow. This chapter dissects every component of the modern Transformer block, explains why each design choice was made, and shows exactly how they fit together.

The Complete Block Anatomy

A Transformer block has a deceptively simple structure: two sublayers — attention and feed-forward — separated by normalization and connected by residuals. The magic is in how these pieces compose. Each sublayer does one job: attention lets tokens talk to each other, and the FFN lets each token think by itself. The normalization keeps signals well-behaved, and the residual connections ensure information can flow unimpeded from the first layer to the last.

Pre-Norm Transformer Block (Modern Standard)

The modern Transformer block applies layer normalization before each sublayer, not after. The computation for one block is:

$$\begin{aligned} \mathbf{h} &= \mathbf{x} + \text{Attention}(\text{Norm}(\mathbf{x})) \\ \mathbf{y} &= \mathbf{h} + \text{FFN}(\text{Norm}(\mathbf{h})) \end{aligned}$$

The residual connections add the sublayer outputs directly to the input, creating an unimpeded "highway" through the network. The normalization ensures each sublayer receives well-conditioned input regardless of what the residual stream carries. This ordering — normalize, compute, add — is used in GPT-2, GPT-3, LLaMA, PaLM, Gemini, and nearly all models trained after 2019.

The full pre-norm Transformer block in PyTorch ties together attention, FFN, residual connections, and normalization into a single reusable unit.


import torch
import torch.nn as nn

class TransformerBlock(nn.Module):
    """Pre-norm Transformer block: the fundamental repeating unit."""
    def __init__(self, d_model, n_heads, d_ff, dropout=0.0):
        super().__init__()
        self.norm1 = RMSNorm(d_model)
        self.attn = MultiHeadAttention(d_model, n_heads)
        self.norm2 = RMSNorm(d_model)
        self.ffn = SwiGLU(d_model, d_ff)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Pre-norm: normalize BEFORE each sublayer
        # Residual connection: add input back AFTER each sublayer

        # Sublayer 1: Self-Attention
        residual = x
        x = self.norm1(x)                          # (batch, seq_len, d_model)
        x = self.attn(x, mask=mask)                 # (batch, seq_len, d_model)
        x = self.dropout(x) + residual              # residual connection

        # Sublayer 2: Feed-Forward Network
        residual = x
        x = self.norm2(x)                           # (batch, seq_len, d_model)
        x = self.ffn(x)                             # (batch, seq_len, d_model)
        x = self.dropout(x) + residual              # residual connection

        return x

# LLaMA-7B config: 32 blocks stacked
blocks = nn.ModuleList([
    TransformerBlock(d_model=4096, n_heads=32, d_ff=11008)
    for _ in range(32)
])

Post-Norm vs Pre-Norm

The original 2017 paper used post-norm: $\text{Norm}(\mathbf{x} + \text{Sublayer}(\mathbf{x}))$. This normalizes the combined signal, which can be unstable because the sublayer output may have arbitrary scale before normalization sees it. Training deep post-norm Transformers (40+ layers) without extensive learning rate warmup often diverges.

Pre-norm — $\mathbf{x} + \text{Sublayer}(\text{Norm}(\mathbf{x}))$ — fixes this by ensuring the sublayer always receives standardized input. The tradeoff: the residual stream itself is never normalized, so its scale grows with depth. Empirically, this doesn't hurt; the final layer norm before the output head handles the accumulated scale.

The DeepNorm Variant

DeepNorm (Wang et al., 2022) enables training Transformers with 1,000+ layers by scaling residuals: $\mathbf{x} \cdot \alpha + \text{Sublayer}(\text{Norm}(\mathbf{x}))$, where $\alpha = (2N)^{1/4}$ for $N$ layers. The sublayer weights are initialized with reduced variance $\beta = (8N)^{-1/4}$. This prevents the residual stream from being dominated by sublayer outputs in very deep networks, maintaining a stable signal-to-noise ratio across arbitrary depth.

Why Pre-Norm Is Now Dominant

Pre-norm Transformers train stably without learning rate warmup (though warmup still helps) and converge reliably even at 100+ layers. The gradient flows cleanly through the residual connection at initialization — the sublayer weights are small, so the block approximates the identity function, and gradients pass through unimpeded. For any practical model size, the slight performance gap that post-norm might offer is not worth the engineering complexity of stabilizing it. Every major LLM since GPT-2 uses pre-norm or a variant of it.

Self-Attention in Depth

Chapter 3 presented the attention formula. Here we go deeper: why are there three separate projections? What do attention heads actually learn? And how does the causal mask work at the implementation level?

Why Separate Q, K, V Projections?

A natural question: why not use the same representation for queries and keys? The answer is that different roles require different features. When a pronoun like "it" searches for its antecedent, the query encoding asks "what noun am I referring to?" while the key encoding of each candidate noun says "I am a subject-position noun with these semantic features." These are fundamentally different questions — the pronoun's search criteria and the noun's advertisement are asymmetric. Sharing a single projection would force these two roles into the same feature space, which limits the model's ability to learn flexible retrieval patterns.

The value projection serves a third role entirely: it determines what information gets sent after a match is found. The query-key interaction decides "who to attend to," while the value determines "what to extract." Separating values from keys lets the model attend to one feature (say, syntactic role) while extracting different information (say, semantic content).

QKV Projections: The Mechanics

Each attention sublayer contains four weight matrices: $W_Q, W_K, W_V \in \mathbb{R}^{d_\text{model} \times d_\text{model}}$ for computing queries, keys, and values, and $W_O \in \mathbb{R}^{d_\text{model} \times d_\text{model}}$ for the output projection. Together, these account for $4 \cdot d_\text{model}^2$ parameters per layer. For LLaMA-7B ($d_\text{model} = 4096$), that is about 67 million parameters per attention sublayer.

Attention as Content-Addressable Memory

Think of a librarian (query) searching a card catalog (keys) to pull the right books (values). The query encodes "what am I looking for," the key encodes "what does this token contain," and the value encodes "what information should be sent if this token is selected." The dot product $q_i^\top k_j$ (dot product, yields a scalar) measures the match between what position $i$ wants and what position $j$ offers. The softmax converts these match scores into a retrieval distribution, and the weighted sum over values produces the retrieved content.

This is exactly a differentiable dictionary lookup. A traditional dictionary maps exact keys to values. Attention maps soft keys to values — every key participates in the output, weighted by its similarity to the query. This softness is what makes attention trainable: gradients flow through all query-key-value pathways, letting the model learn which matches are useful.

Attention Patterns in Trained Models

Visualization of trained attention heads reveals distinct pattern types that emerge without explicit supervision. Understanding these patterns helps build intuition about what the model is doing at each layer.

  • Local patterns: attending primarily to the token itself and its immediate neighbors — common in early layers that build local context.
  • Global patterns: attending broadly across the sequence — common in middle layers performing long-range reasoning.
  • Diagonal patterns: attending to the same relative position across all query positions — encoding fixed positional relationships like "attend to the previous token."
  • Vertical stripes: all positions attending strongly to a few "sentinel" tokens (often punctuation, [CLS], or high-frequency words) — these tokens serve as global information hubs.

Causal Masking Implementation

For decoder-only models (GPT, LLaMA), each token must only attend to tokens at its position or earlier — never to the future. Without this constraint, the model would trivially learn to "copy the answer" from future tokens instead of predicting it. The implementation creates a lower-triangular boolean mask and adds $-\infty$ (in practice, a large negative number like $-10^9$) to the masked positions before softmax. After softmax, $e^{-\infty} = 0$, so future positions contribute zero weight.


import torch

def causal_mask(seq_len: int) -> torch.Tensor:
    """Create a causal (lower-triangular) attention mask."""
    # True = allowed to attend, False = masked
    mask = torch.tril(torch.ones(seq_len, seq_len, dtype=torch.bool))
    return mask

# Usage in attention:
# scores shape: (batch, heads, seq_len, seq_len)
mask = causal_mask(seq_len).to(scores.device)
scores = scores.masked_fill(~mask, float('-inf'))
attn_weights = torch.softmax(scores, dim=-1)

Multi-Head Attention with Einsum

The most efficient implementation uses fused QKV projections and torch.nn.functional.scaled_dot_product_attention, which dispatches to FlashAttention when available. The manual version using einsum is more readable and pedagogically useful:


import torch
from einops import rearrange

def multi_head_attention(x, W_qkv, W_o, n_heads, mask=None):
    """Multi-head attention using einops for clarity."""
    B, N, D = x.shape
    d_k = D // n_heads

    # Fused QKV projection: single matmul is faster than three
    qkv = x @ W_qkv  # (B, N, 3*D)
    q, k, v = rearrange(qkv, 'b n (three h d) -> three b h n d',
                         three=3, h=n_heads)

    # Scaled dot-product attention
    scores = torch.einsum('bhid,bhjd->bhij', q, k) / (d_k ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(~mask, float('-inf'))
    attn = scores.softmax(dim=-1)
    out = torch.einsum('bhij,bhjd->bhid', attn, v)

    # Concatenate heads and project
    out = rearrange(out, 'b h n d -> b n (h d)')
    return out @ W_o

The two einsum operations are the core of attention. The first — 'bhid,bhjd->bhij' — computes pairwise dot products between all query-key pairs within each head. The second — 'bhij,bhjd->bhid' — weighted-sums the values according to the attention distribution. Together with the $1/\sqrt{d_k}$ scaling and the softmax, these two operations implement the complete attention formula.

Feed-Forward Sublayer in Depth

The attention sublayer lets tokens communicate. The FFN sublayer lets each token compute. While attention is the Transformer's defining innovation, the FFN accounts for the majority of parameters and is where factual knowledge gets stored. Understanding its structure — and the evolution from ReLU to SwiGLU — is essential for understanding modern model design.

The Two-Layer MLP

The feed-forward sublayer applies the same two-layer MLP independently to every token position. With ReLU activation, the computation is:

$$\text{FFN}(\mathbf{x}) = \text{ReLU}(\mathbf{x}W_1 + \mathbf{b}_1)W_2 + \mathbf{b}_2$$

The up-projection $\mathbf{x}W_1$ (matrix multiply, $\mathbb{R}^{n \times d_\text{model}} \times \mathbb{R}^{d_\text{model} \times d_\text{ff}} \to \mathbb{R}^{n \times d_\text{ff}}$) expands from $d_\text{model}$ to $d_\text{ff}$ (typically $4 \times d_\text{model}$), and the down-projection $W_2 \in \mathbb{R}^{d_\text{ff} \times d_\text{model}}$ compresses back. The expansion creates a high-dimensional "workspace" where the network can represent more complex functions before projecting back to the residual dimension.

SwiGLU: The Modern Default

SwiGLU replaces the simple up-projection + activation with a gated mechanism. Instead of one up-projection, there are two: one computes a gate, the other computes the content. The gate controls how much of each hidden dimension passes through:

$$\text{SwiGLU}(\mathbf{x}) = (\text{SiLU}(\mathbf{x}W_\text{gate}) \odot \mathbf{x}W_\text{up}) \, W_\text{down}$$

where $\text{SiLU}(x) = x \cdot \sigma(x)$ is the Swish/SiLU activation and $\odot$ denotes element-wise multiplication. The gate projection $W_\text{gate}$ and the up projection $W_\text{up}$ both map from $d_\text{model}$ to $d_\text{ff}$, and $W_\text{down}$ maps back. Three matrices instead of two means 50% more parameters at the same $d_\text{ff}$, which is why SwiGLU models use $d_\text{ff} \approx \frac{8}{3} d_\text{model}$ to stay parameter-equivalent.

Here is SwiGLU in PyTorch, matching the architecture used in LLaMA and PaLM:


import torch
import torch.nn as nn
import torch.nn.functional as F

class SwiGLU(nn.Module):
    """SwiGLU feed-forward network as used in LLaMA."""
    def __init__(self, d_model: int, d_ff: int):
        super().__init__()
        # Three projections: gate, up, and down
        self.w_gate = nn.Linear(d_model, d_ff, bias=False)
        self.w_up   = nn.Linear(d_model, d_ff, bias=False)
        self.w_down = nn.Linear(d_ff, d_model, bias=False)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # Gate: SiLU activation on one projection
        gate = F.silu(self.w_gate(x))
        # Up: linear projection (no activation)
        up = self.w_up(x)
        # Element-wise product, then project back
        return self.w_down(gate * up)

# LLaMA-7B: d_model=4096, d_ff=11008 ≈ 8/3 × 4096
ffn = SwiGLU(d_model=4096, d_ff=11008)

Why SwiGLU Works

The gating mechanism allows the network to learn input-dependent feature selection. Rather than applying a fixed nonlinearity to every dimension, the gate dynamically decides which dimensions to activate for each input. Think of it as two parallel pathways: one computes "what features are present" (the up-projection) while the other computes "which features matter right now" (the gate). The element-wise product combines them: only features that are both present and deemed relevant survive.

This is smoother and more expressive than ReLU's binary on/off behavior. Shazeer (2020) showed that SwiGLU consistently outperforms ReLU, GELU, and other activations across model sizes, with the improvement becoming more pronounced at larger scale.

FFN as Key-Value Memory

Geva et al. (2021) demonstrated that FFN layers function as key-value memories. Each row of $W_1$ (or $W_\text{gate}$) acts as a "key" that detects a specific input pattern. When activated, the corresponding column of $W_2$ (or $W_\text{down}$) provides the "value" — an update vector that gets added to the residual stream. This interpretation explains why FFN layers store factual knowledge: the key-value pairs encode associations like "if the input matches the pattern [Eiffel Tower located in], then write [Paris] into the residual stream."

This view has practical implications. Editing specific FFN weights can surgically modify a model's beliefs — changing a single key-value pair can update a factual association without affecting unrelated knowledge. Techniques like ROME (Rank-One Model Editing) exploit exactly this structure.

FFN vs Attention Parameter Count

At typical ratios, the FFN sublayer contains more parameters than the attention sublayer. For standard attention with $d_\text{model} = 4096$:

Component Matrices Parameters
Attention (QKV + O) $4 \times d_\text{model}^2$ 67M
FFN with ReLU ($4\times$) $2 \times d_\text{model} \times 4 d_\text{model}$ 134M
FFN with SwiGLU ($\frac{8}{3}\times$) $3 \times d_\text{model} \times \frac{8}{3} d_\text{model}$ 134M

The FFN accounts for roughly two-thirds of each block's parameters. This is why techniques like Mixture of Experts (MoE) target the FFN specifically — it is the largest parameter bottleneck and the most natural place to add conditional computation.

Residual Connections

Residual connections are the simplest component in the Transformer block — just addition — yet they are arguably the most important for enabling depth. Without them, Transformers beyond a handful of layers would be untrainable.

Original Motivation

He et al. (2016) introduced residual connections for deep CNNs, solving the degradation problem: adding more layers to a network made it worse, not better. The residual connection $\mathbf{y} = \mathbf{x} + F(\mathbf{x})$ provides a shortcut: if the sublayer $F$ should ideally compute the identity (do nothing), it only needs to learn $F(\mathbf{x}) = 0$, which is easy. Without the shortcut, learning the identity through a nonlinear $F$ is surprisingly hard.

Identity Initialization and Gradient Flow

At initialization, all sublayer weights are small (drawn from a distribution with small variance), so $F(\mathbf{x}) \approx 0$. This means the entire Transformer starts as an approximate identity function: each block passes its input straight through. This property is critical for training stability — it means that at the start of training, gradients flow cleanly through the entire network via the residual connections, before the sublayers have learned anything useful.

During backpropagation, the gradient of the loss with respect to an early layer's input includes a direct term (through the residual connection) plus terms through each sublayer. Even if the sublayer gradients vanish or explode, the direct term ensures some gradient signal always reaches early layers. Formally, the Jacobian of the residual block is $\frac{\partial \mathbf{y}}{\partial \mathbf{x}} = I + \frac{\partial F(\mathbf{x})}{\partial \mathbf{x}}$, where the identity matrix $I$ guarantees a gradient magnitude of at least 1 in every direction. This is why Transformers can be trained at depths (100+ layers) that are difficult for other architectures.

The Residual Stream Interpretation

The mechanistic interpretability community has reframed how we think about residual connections. Instead of viewing each layer as a transformation, view the residual connections as a residual stream — a running vector that each sublayer reads from and writes to. The stream starts as the token embedding plus positional encoding. Each attention sublayer reads the stream, computes something, and adds its result back. Each FFN sublayer does the same. After $L$ layers, the stream contains the original embedding plus the sum of all sublayer outputs:

$$\mathbf{x}_L = \mathbf{x}_0 + \sum_{\ell=1}^{L} \text{Attn}_\ell(\cdot) + \sum_{\ell=1}^{L} \text{FFN}_\ell(\cdot)$$

This view is powerful because it implies that each sublayer's contribution is additive. You can study each sublayer's output independently and understand what it adds to the stream. Ablating a single sublayer (setting its output to zero) removes exactly one term from this sum, making it straightforward to measure that sublayer's contribution to any particular behavior.

Residual Stream as Superposition

The residual stream at dimension $d_\text{model}$ carries far more features than it has dimensions. Anthropic's work on superposition (Elhage et al., 2022) showed that models encode many nearly-orthogonal features into the same $d_\text{model}$-dimensional space. The residual stream is not a clean set of dedicated feature slots — it is a compressed representation where features interfere with each other, but the interference is kept small enough that downstream layers can still read the features they need. This insight — that the stream holds a superposition of many more features than its dimensionality suggests — has profound implications for interpretability and helps explain why scaling $d_\text{model}$ steadily improves performance.

Putting everything together, here is a complete decoder-only language model that stacks embeddings, N Transformer blocks, and an output projection into an end-to-end architecture.


class DecoderLM(nn.Module):
    """Complete decoder-only language model: embeddings → N blocks → output logits."""
    def __init__(self, vocab_size, d_model, n_heads, d_ff, n_layers, max_len=2048):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff)
            for _ in range(n_layers)
        ])
        self.final_norm = RMSNorm(d_model)
        # Tie output projection with token embeddings
        self.output = nn.Linear(d_model, vocab_size, bias=False)
        self.output.weight = self.token_emb.weight   # weight tying

    def forward(self, input_ids, mask=None):
        B, S = input_ids.shape
        positions = torch.arange(S, device=input_ids.device)

        # Embed tokens + positions
        x = self.token_emb(input_ids) + self.pos_emb(positions)  # (B, S, d_model)

        # Pass through N transformer blocks
        for block in self.blocks:
            x = block(x, mask=mask)                  # (B, S, d_model)

        # Final norm + project to vocabulary
        x = self.final_norm(x)                       # (B, S, d_model)
        logits = self.output(x)                      # (B, S, vocab_size)
        return logits

# GPT-2 Small equivalent: 12 layers, 768 dim, 12 heads
model = DecoderLM(vocab_size=50257, d_model=768, n_heads=12, d_ff=3072, n_layers=12)
print(f"Parameters: {sum(p.numel() for p in model.parameters()):,}")  # ~124M

Normalization Variants

Normalization is the unsung hero of deep Transformer training. Without it, the activations in deep networks drift to extreme values, gradients explode or vanish, and training diverges. The question is not whether to normalize, but how and where.

Layer Norm: The Foundation

Standard Layer Norm (Ba et al., 2016) computes mean and variance across the feature dimension for each token independently, then applies a learned affine transformation:

$$\text{LayerNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

where $\mu = \frac{1}{d}\sum_{i=1}^d x_i$ is the mean, $\sigma^2 = \frac{1}{d}\sum_{i=1}^d (x_i - \mu)^2$ is the variance, and $\gamma, \beta \in \mathbb{R}^d$ are learned scale and shift parameters. The small constant $\epsilon$ (typically $10^{-5}$) prevents division by zero. Layer Norm has $2 \times d_\text{model}$ learnable parameters. The normalization ensures that each sublayer receives inputs with consistent scale, regardless of what the residual stream has accumulated.

Why Layer Norm and not Batch Norm? Batch Norm computes statistics across the batch, which fails for variable-length sequences (different positions would have different batch statistics) and breaks at inference time with batch size 1. Layer Norm computes statistics within each token, making it independent of batch size and sequence length.

RMS Norm: Simpler and Faster

RMS Norm (Zhang and Sennrich, 2019) drops the mean centering, normalizing only by the root mean square:

$$\text{RMSNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x}}{\text{RMS}(\mathbf{x})} \quad\text{where}\quad \text{RMS}(\mathbf{x}) = \sqrt{\frac{1}{d}\sum_{i=1}^d x_i^2}$$

It has only $d_\text{model}$ parameters (just $\gamma$, no $\beta$ shift). This is both faster (one fewer reduction operation — no mean computation, no subtraction) and empirically equivalent. The insight is that re-centering activations to zero mean provides marginal benefit, while re-scaling to unit variance does the heavy lifting. LLaMA, PaLM, Gemini, and Mistral all use RMS Norm.

Here is a clean PyTorch implementation of RMSNorm, matching the version used in LLaMA and Gemma.


import torch
import torch.nn as nn

class RMSNorm(nn.Module):
    """Root Mean Square Layer Normalization (used in LLaMA, Gemma)."""
    def __init__(self, d_model, eps=1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(d_model))  # learnable scale γ
        self.eps = eps

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        x_normed = x / rms                     # normalize by RMS
        return self.weight * x_normed           # element-wise scale (broadcast)

Pre-Norm: Training Stability Without Warmup

The placement of normalization matters as much as the type. Pre-norm (normalize before each sublayer) ensures that the sublayer always sees inputs with bounded scale. At initialization, where sublayer weights are small, the block output is approximately $\mathbf{x} + \epsilon \cdot \text{Sublayer}(\text{Norm}(\mathbf{x}))$, which is close to the identity. This means:

  • The network behaves like the identity at init, so gradients flow cleanly from the start.
  • No warmup phase is needed to stabilize training (though warmup still helps for final quality).
  • Very deep models (100+ layers) train without divergence.

The cost is that the residual stream accumulates unnormalized updates, so its scale grows with depth. A final layer norm before the output projection compensates for this.

Deep Norm: Scaling to 1,000+ Layers

For extremely deep Transformers (1,000+ layers), even pre-norm can struggle. Deep Norm (Wang et al., 2022) uses a scaled residual: $\alpha \mathbf{x} + \text{Sublayer}(\text{Norm}(\mathbf{x}))$ with $\alpha = (2N)^{1/4}$ for $N$ layers, combined with reduced initialization variance $\beta = (8N)^{-1/4}$ for the sublayer weights. This keeps the residual stream's scale stable across arbitrary depth, enabling 1,000-layer Transformers to train without instability.

Activation Functions Survey

The activation function in the FFN sublayer has a larger impact on model quality than you might expect. Each generation of Transformer models has adopted a better activation, and the progression tells a clear story about what properties matter.

ReLU: Sparse but Brittle

ReLU ($\max(0, x)$) produces sparse activations — typically 50% of neurons are zero at any given input. Sparsity can be a feature (interpretability, efficiency), but the dead neuron problem is a liability. Once a neuron's pre-activation becomes permanently negative (due to a large negative bias learned during training), it produces zero output and zero gradient forever. In large Transformers, up to 10% of ReLU neurons can become permanently dead.

GELU: Smooth Approximation

GELU approximates ReLU but with a smooth curve: $\text{GELU}(x) = x \cdot \Phi(x)$ where $\Phi$ is the Gaussian CDF. Small negative inputs get a small negative output instead of exactly zero, so gradients always flow. BERT and GPT-2 adopted GELU, and it became the default for several years.

SiLU/Swish: Self-Gated

SiLU (Sigmoid Linear Unit), also known as Swish, is $x \cdot \sigma(x)$ where $\sigma$ is the sigmoid function. Like GELU, it is smooth everywhere and allows small negative outputs. The self-gating property — where $\sigma(x)$ acts as a learnable gate on $x$ — foreshadows the explicit gating in SwiGLU. SiLU serves as the gating function inside SwiGLU.

SwiGLU and GeGLU: Gated Linear Units

SwiGLU uses SiLU as the gate: $\text{SiLU}(xW_\text{gate}) \odot xW_\text{up}$. GeGLU uses GELU instead: $\text{GELU}(xW_\text{gate}) \odot xW_\text{up}$. Both outperform their ungated counterparts. SwiGLU is the current winner and is used in LLaMA, PaLM, Gemini, and Mistral. The gating mechanism provides input-dependent feature selection — a form of dynamic routing that is more powerful than static nonlinearities.

Key Insight

The evolution from ReLU → GELU → SwiGLU follows a clear pattern: each successor is smoother, allows more gradient flow, and provides more adaptive nonlinearity. The consistent lesson is that hard thresholds (ReLU's zero cutoff) hurt optimization, and input-dependent gating (SwiGLU) helps the model allocate its FFN capacity more efficiently across different inputs.
