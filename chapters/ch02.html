Chapter 02

Mathematical Prerequisites

Every operation inside a Transformer — self-attention, feed-forward layers, normalization, output projection — reduces to a handful of mathematical building blocks: matrix multiplications, a particular normalization function called softmax, and a specific way of standardizing activations called layer norm. Misunderstanding any one of these blocks leads to confusion about everything built on top of them. By the end of this chapter, you will not only know the formulas but understand why each mathematical choice was made: why we scale dot products, why layer norm replaced batch norm, why the feed-forward network uses a 4× expansion, and why activation functions like SwiGLU outperform ReLU.

Linear Algebra Review for Attention

Matrix Multiplication as the Core Operation

A Transformer processes sequences of tokens, and each token is a vector of numbers — its embedding. When we stack all token embeddings from a sequence into a matrix, every Transformer operation becomes a matrix multiplication. The projection from tokens to queries is a matrix multiply. The attention score computation is a matrix multiply. The feed-forward layer is two matrix multiplies with a nonlinearity in between. Understanding Transformers means understanding what matrix multiplication does geometrically and computationally.

Consider a sequence of $n$ tokens, each represented as a $d$-dimensional vector. We stack them into a matrix $X \in \mathbb{R}^{n \times d}$, where each row is a token. Multiplying by a weight matrix $W \in \mathbb{R}^{d \times d'}$ produces $Y = XW$ (matrix multiply, $\mathbb{R}^{n \times d} \times \mathbb{R}^{d \times d'} \to \mathbb{R}^{n \times d'}$). Each row of $Y$ is a linear combination of the columns of $W$, weighted by the corresponding token's embedding values. The weight matrix $W$ defines a learned linear transformation that remaps every token into a new representational space.

Vectors as Token Representations, Matrices as Transformations

Think of each token embedding as a point in a high-dimensional space. The word "king" occupies one point, "queen" another, "apple" a third. A weight matrix $W$ rotates, stretches, and projects these points into a new coordinate system. When we compute $Q = XW_Q$, we are asking: "given these token embeddings, what does each token want to find?" The matrix $W_Q$ is a learned rotation that maps the raw embedding space into "query space" — a coordinate system optimized for searching.

The Role of Dimensionality

Two dimensions govern the Transformer's capacity. The model dimension $d_\text{model}$ is the width of every token representation throughout the network — in GPT-3, this is 12,288. The head dimension $d_k$ is the width of queries and keys inside each attention head, typically $d_\text{model} / h$ where $h$ is the number of attention heads. A larger $d_\text{model}$ gives each token more expressive capacity, while a smaller $d_k$ per head forces each head to specialize on a lower-dimensional subspace.

Why not just use one giant attention head? Because splitting into multiple heads lets the model attend to different types of relationships simultaneously — one head might track syntactic dependencies while another tracks coreference. The split is free: the total parameter count is the same whether you use 1 head of dimension 768 or 12 heads of dimension 64.

Outer Products, Dot Products, and Their Geometric Interpretation

The dot product between two vectors $\mathbf{a}, \mathbf{b} \in \mathbb{R}^d$ is the scalar $\mathbf{a}^\top \mathbf{b} = \sum_{i=1}^d a_i b_i$ (dot product, yields a scalar). Geometrically, it equals $\|\mathbf{a}\| \|\mathbf{b}\| \cos\theta$, where $\theta$ is the angle between them. When vectors point in the same direction, the dot product is large and positive. When they are orthogonal, it is zero. This is exactly the property that attention exploits: the dot product between a query and a key measures how "aligned" they are — how much the query is looking for what the key represents.

The outer product $\mathbf{a}\mathbf{b}^\top$ produces a $d \times d$ matrix where entry $(i,j) = a_i b_j$. This appears less obviously in the Transformer, but it plays a key role in understanding how attention aggregates information. The attention-weighted sum $\sum_j \alpha_j \mathbf{v}_j$ can be rewritten as the attention matrix (which is built from outer-product-like operations between query and key vectors) multiplied by the value matrix. Keeping both products — inner and outer — in mind helps you read any Transformer equation fluently.

The Softmax Function

Definition and Properties

The softmax function takes a vector of raw scores (logits) and converts them into a probability distribution — non-negative values that sum to one. Every attention head uses softmax to convert raw query-key dot products into attention weights. Every language model uses softmax in its final layer to produce a distribution over the vocabulary. It is the single most important nonlinearity in the Transformer.

Given a vector $\mathbf{z} \in \mathbb{R}^n$, softmax maps each element as:

$$\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^n e^{z_j}}$$

The numerator $e^{z_i}$ ensures every output is positive. The denominator normalizes so all outputs sum to one. The function is differentiable everywhere, which is essential for gradient-based training. It is also monotonic: if $z_i > z_j$, then $\text{softmax}(z_i) > \text{softmax}(z_j)$. The relative ordering of inputs is always preserved.

Numerical Stability

A naive implementation is dangerous. If any $z_i$ is large (say 1000), computing $e^{1000}$ overflows to infinity in floating-point arithmetic. The standard fix is to subtract the maximum value before exponentiating:

$$\text{softmax}(z_i) = \frac{e^{z_i - \max(\mathbf{z})}}{\sum_{j=1}^n e^{z_j - \max(\mathbf{z})}}$$

This is mathematically identical — the $e^{-\max(\mathbf{z})}$ factor cancels between numerator and denominator — but now the largest exponent is $e^0 = 1$, which never overflows. Every production implementation of softmax uses this trick, and forgetting it causes silent NaN errors in training.


import numpy as np

def softmax_naive(z):
    """Numerically unstable — will produce inf for large inputs."""
    exp_z = np.exp(z)
    return exp_z / exp_z.sum()

def softmax_stable(z):
    """Numerically stable — subtracts max before exponentiating."""
    z_shifted = z - np.max(z)
    exp_z = np.exp(z_shifted)
    return exp_z / exp_z.sum()

# Demonstration
logits = np.array([1000.0, 1001.0, 1002.0])
print(softmax_naive(logits))   # [nan, nan, nan]
print(softmax_stable(logits))  # [0.0900, 0.2447, 0.6652]

Temperature Scaling

Temperature controls how "peaked" the output distribution is. Instead of computing $\text{softmax}(\mathbf{z})$ directly, we compute $\text{softmax}(\mathbf{z}/\tau)$ where $\tau > 0$ is the temperature. Low temperature ($\tau \to 0$) drives the distribution toward a hard argmax — one element gets nearly all the probability mass. High temperature ($\tau \to \infty$) flattens the distribution toward uniform. In attention, the $1/\sqrt{d_k}$ scaling factor acts exactly like a temperature: it prevents the dot products from growing so large that softmax saturates.

Here is temperature-scaled softmax in PyTorch, showing how the same logits produce sharply different distributions:


import torch
import torch.nn.functional as F

logits = torch.tensor([2.0, 5.0, 1.0, 3.0, 4.0])  # (5,)

for tau in [0.1, 0.5, 1.0, 2.0, 5.0]:
    probs = F.softmax(logits / tau, dim=0)           # (5,) — same shape, different sharpness
    top_prob = probs.max().item()
    entropy = -(probs * probs.log()).sum().item()
    print(f"τ={tau:<4}  probs={probs.numpy().round(3)}  max={top_prob:.3f}  H={entropy:.2f}")

# τ=0.1   probs=[0.    1.    0.    0.    0.   ]  max=1.000  H=0.00  ← near-argmax
# τ=0.5   probs=[0.002 0.953 0.    0.018 0.027]  max=0.953  H=0.24
# τ=1.0   probs=[0.047 0.643 0.018 0.117 0.175]  max=0.643  H=1.16
# τ=2.0   probs=[0.104 0.373 0.073 0.174 0.276]  max=0.373  H=1.46
# τ=5.0   probs=[0.150 0.247 0.130 0.192 0.281]  max=0.281  H=1.57  ← near-uniform

Softmax Saturation and Gradient Vanishing

When logits become very large in magnitude, the softmax output approaches a one-hot vector — one element near 1.0, the rest near 0.0. In this saturated regime, gradients vanish. The winning element has gradient near zero (its output barely changes with small logit changes), and the losing elements have exponentially small gradients. Training stalls.

This is precisely why the $1/\sqrt{d_k}$ scaling factor in attention exists. Without it, dot products between $d_k$-dimensional vectors have variance proportional to $d_k$ (because each of the $d_k$ terms in the sum contributes variance). For $d_k = 64$, unscaled dot products would have standard deviation roughly 8, putting most attention distributions deep in the saturated regime. Dividing by $\sqrt{d_k}$ restores unit variance, keeping the softmax in the regime where gradients flow.

Layer Normalization

Batch Norm vs Layer Norm

Before Transformers, batch normalization was the dominant technique for stabilizing deep network training. Batch norm computes the mean and variance across the batch dimension — for each feature, it averages over all examples in the mini-batch. This works beautifully for fixed-size inputs like images. For sequences, it breaks down.

The problem is twofold. First, sequences in a batch have different lengths, so averaging "the value at position 47" across a batch mixes real tokens with padding tokens, polluting the statistics. Second, at inference time, batch norm requires running-average statistics accumulated during training, but these statistics are meaningless when the test sequence length differs from training lengths. Layer normalization avoids both problems by normalizing across the feature dimension within a single example, making it completely independent of batch composition and sequence length.

Layer Norm Formula

Given a single vector $\mathbf{x} \in \mathbb{R}^d$ (one token's representation), layer norm computes:

$$\text{LayerNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

where $\mu = \frac{1}{d}\sum_{i=1}^d x_i$ is the mean across features, $\sigma^2 = \frac{1}{d}\sum_{i=1}^d (x_i - \mu)^2$ is the variance, $\epsilon \approx 10^{-5}$ prevents division by zero, and $\gamma, \beta \in \mathbb{R}^d$ are learned scale and shift parameters. The $\odot$ denotes element-wise multiplication (Hadamard product, both $\in \mathbb{R}^{d}$). The normalization centers and standardizes the activations, then $\gamma$ and $\beta$ allow the network to undo the normalization for any dimensions where it hurts — giving the model the option of ignoring the norm if that is optimal.

RMS Norm

RMS Norm (Root Mean Square Normalization) simplifies layer norm by removing the mean-centering step. Instead of computing both mean and variance, it normalizes by the root mean square alone:

$$\text{RMSNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x}}{\sqrt{\frac{1}{d}\sum_{i=1}^d x_i^2 + \epsilon}}$$

The bias $\beta$ is also dropped. This saves a small amount of computation (one fewer reduction across the feature dimension) and empirically works just as well. LLaMA, Gemma, and most modern large language models use RMS Norm exclusively. The intuition is that re-centering activations to have zero mean provides negligible benefit once the model is large enough — what matters is controlling the overall scale.


import torch
import torch.nn as nn

class RMSNorm(nn.Module):
    def __init__(self, d_model: int, eps: float = 1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(d_model))  # learnable scale γ
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x shape: (batch, seq_len, d_model)
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return self.weight * (x / rms)

Pre-Norm vs Post-Norm

The original Transformer applied layer norm after the residual addition: $\text{LayerNorm}(x + \text{Sublayer}(x))$. This is called post-norm. It works, but training deep post-norm Transformers (40+ layers) without careful learning rate warmup often diverges. The reason: the residual connection adds the sublayer output to $x$ before normalization, so the scale of the combined signal can grow without bound during early training.

Pre-norm applies normalization before the sublayer: $x + \text{Sublayer}(\text{LayerNorm}(x))$. The sublayer always receives a well-conditioned input, regardless of what the residual stream looks like. This dramatically improves training stability and is now the default in nearly all large Transformers (GPT-2 onward, LLaMA, PaLM, Gemini). The tradeoff is subtle: some work suggests pre-norm produces slightly worse final loss for the same parameter count, because the unnormalized residual stream can accumulate information more freely. In practice, the stability benefit dominates.

Scaled Dot-Product Attention Derivation

Why Scale by $\sqrt{d_k}$

Imagine two random vectors $\mathbf{q}, \mathbf{k} \in \mathbb{R}^{d_k}$ where each component is drawn independently from a distribution with mean 0 and variance 1. Their dot product is $\sum_{i=1}^{d_k} q_i k_i$ — a sum of $d_k$ independent terms, each with mean 0 and variance 1. By the properties of sums of independent random variables, the dot product has mean 0 and variance $d_k$, giving a standard deviation of $\sqrt{d_k}$.

For $d_k = 64$, the standard deviation is 8. Dot products will routinely land at values like $\pm 16$ or beyond. Fed into softmax, these produce distributions that are nearly one-hot — one key gets weight 0.9999, the rest share 0.0001. The gradients are effectively zero, and the model cannot learn which tokens to attend to. Dividing by $\sqrt{d_k}$ rescales the dot products to have unit variance, keeping softmax in its sensitive regime where gradients are informative.

The Full Attention Formula

Now let us write down exactly what we described. Given queries $Q \in \mathbb{R}^{n \times d_k}$, keys $K \in \mathbb{R}^{n \times d_k}$, and values $V \in \mathbb{R}^{n \times d_v}$:

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$

The $QK^\top$ term computes an $n \times n$ matrix of pairwise similarity scores between every query and every key — entry $(i,j)$ is how much token $i$'s query aligns with token $j$'s key. Dividing by $\sqrt{d_k}$ prevents the scores from saturating softmax. The softmax (applied row-wise) converts each row into a probability distribution over positions. Multiplying by $V$ produces a weighted average of value vectors, where the weights are the attention probabilities. Each output row is a "summary" of the sequence, weighted by relevance to that query.

Complexity Analysis

The $QK^\top$ multiplication is the bottleneck: multiplying an $(n \times d_k)$ matrix by a $(d_k \times n)$ matrix costs $O(n^2 d_k)$ time and produces an $(n \times n)$ attention matrix that requires $O(n^2)$ memory. For a sequence of 4,096 tokens with $d_k = 64$, the attention matrix alone contains 16.7 million entries per head. With 32 heads, that is 537 million entries per layer. This quadratic scaling in sequence length is the fundamental computational bottleneck of the Transformer — and the motivation behind FlashAttention, sparse attention, and all the efficiency innovations we cover in Part VI.

Gradient Flow Through the Softmax

The Jacobian of softmax with respect to its input has a specific structure: $\frac{\partial \text{softmax}(z_i)}{\partial z_j} = \text{softmax}(z_i)(\delta_{ij} - \text{softmax}(z_j))$, where $\delta_{ij}$ is the Kronecker delta. When the distribution is uniform, the Jacobian entries are all roughly $1/n$ in magnitude — gradients flow evenly to all logits. When the distribution is peaked (one entry dominates), the diagonal term is near zero for the winner (since $\text{softmax}(z_i)(1 - \text{softmax}(z_i)) \approx 0$ when $\text{softmax}(z_i) \approx 1$), and off-diagonal terms are also near zero. This is the precise mechanism of gradient vanishing under saturation.

This gradient structure explains why initialization matters. If the query and key projection weights are initialized with too-large variance, early dot products will be large, softmax will saturate, and the attention layers will be effectively frozen during the first phase of training. Standard practice initializes $W_Q$ and $W_K$ with variance $1/d_\text{model}$, keeping early dot products small.

Key Insight

The $1/\sqrt{d_k}$ scaling and the initialization variance of $W_Q, W_K$ serve the same goal from different angles: keeping the softmax in its linear regime where gradients flow. Scaling fixes the issue at every forward pass; initialization fixes it at the start of training. Both are necessary because training dynamics can push dot products back toward saturation.

Feed-Forward Network Mathematics

Two-Layer MLP Structure

After attention aggregates information across the sequence, each token passes independently through a position-wise feed-forward network (FFN). "Position-wise" means the same network is applied to every token separately — no inter-token communication happens here. The FFN is a simple two-layer MLP:

$$\text{FFN}(\mathbf{x}) = W_2 \, \sigma(W_1 \mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2$$

where $W_1 \in \mathbb{R}^{d_\text{ff} \times d_\text{model}}$ projects up to a wider hidden dimension, $\sigma$ is a nonlinear activation function, and $W_2 \in \mathbb{R}^{d_\text{model} \times d_\text{ff}}$ projects back down to the model dimension. The input $\mathbf{x} \in \mathbb{R}^{d_\text{model}}$ enters and exits at the same dimensionality, but passes through a higher-dimensional intermediate representation.

The Expansion Ratio

The standard expansion ratio is $d_\text{ff} = 4 \times d_\text{model}$. For a model with $d_\text{model} = 768$, the FFN hidden dimension is 3,072. Why 4×? The original paper chose it empirically, but there is an intuitive justification: the FFN needs enough capacity to store and retrieve factual knowledge. Work by Geva et al. (2021) showed that FFN layers act as key-value memories — the first layer's rows are "keys" that match input patterns, and the second layer's columns are "values" that produce the corresponding output. A 4× expansion means four times as many memory slots as the model dimension, providing a substantial factual knowledge capacity.

Modern models sometimes use different ratios. LLaMA uses $d_\text{ff} = \frac{8}{3} d_\text{model}$ (rounded to the nearest multiple of 256) when using SwiGLU, because the gating mechanism adds a third weight matrix, and the reduced ratio keeps total parameter count comparable to the standard 4× with ReLU.

Activation Functions: ReLU, GELU, SwiGLU, GLU Variants

The original Transformer used ReLU: $\text{ReLU}(x) = \max(0, x)$. Simple and efficient, but it creates "dead neurons" — any unit with a negative pre-activation contributes exactly zero gradient and may never recover.

GELU (Gaussian Error Linear Unit) replaced ReLU in BERT and GPT-2: $\text{GELU}(x) = x \cdot \Phi(x)$, where $\Phi$ is the standard Gaussian CDF. Unlike ReLU, GELU is smooth everywhere and gives small negative inputs a small nonzero output, avoiding dead neurons. It can be approximated as $\text{GELU}(x) \approx 0.5x(1 + \tanh[\sqrt{2/\pi}(x + 0.044715x^3)])$.

SwiGLU is the current state-of-the-art, used in LLaMA, PaLM, and Gemini. It modifies the FFN structure itself by introducing a gating mechanism:

$$\text{SwiGLU}(\mathbf{x}) = (\text{Swish}(W_1 \mathbf{x}) \odot W_3 \mathbf{x}) W_2$$

where $\text{Swish}(x) = x \cdot \sigma(x)$ with $\sigma$ being the sigmoid function, $\odot$ is element-wise multiplication, and $W_3$ is a third weight matrix of the same shape as $W_1$. The gate $\text{Swish}(W_1 \mathbf{x})$ decides how much of each dimension to keep, while $W_3 \mathbf{x}$ provides the actual content. This gating costs 50% more parameters than standard FFN (three matrices instead of two), which is why models using SwiGLU reduce $d_\text{ff}$ to $\frac{8}{3} d_\text{model}$ to stay parameter-equivalent.

SwiGLU belongs to the broader Gated Linear Unit (GLU) family introduced by Dauphin et al. (2017). The general form is:

$$\text{GLU}(\mathbf{x}) = \sigma_g(W_1 \mathbf{x}) \odot W_3 \mathbf{x}$$

where $\sigma_g$ is a gate activation function. The choice of $\sigma_g$ defines each variant: sigmoid gives the original GLU, ReLU gives ReGLU, GELU gives GeGLU, and Swish gives SwiGLU. Shazeer (2020) systematically benchmarked all variants and found SwiGLU and GeGLU consistently outperformed the alternatives on language modeling tasks, with SwiGLU holding a slight edge. The gating mechanism allows the network to learn *which* dimensions of the intermediate representation to suppress, providing a richer inductive bias than a pointwise activation alone.

Universal Approximation: FFN as a Key-Value Memory Store

Geva et al. (2021) provided a compelling reinterpretation of what FFN layers actually do. Consider $W_1$ as a matrix whose rows are "key patterns" and $W_2$ as a matrix whose columns are "value vectors." The first multiplication $W_1 \mathbf{x}$ computes a dot product between the input and each row of $W_1$, producing an activation score for each row — high when the input matches that key pattern, low otherwise. After the nonlinearity zeroes out the non-matching keys, the second multiplication $W_2$ reads out the value vectors corresponding to the surviving keys.

This view explains why FFN layers store factual knowledge. When a token embedding activates the row of $W_1$ corresponding to "capital of France," the corresponding column in $W_2$ contributes the output direction associated with "Paris." The FFN is literally a differentiable lookup table with $d_\text{ff}$ slots, and training fills those slots with useful associations. This also explains why larger $d_\text{ff}$ (more slots) improves factual recall, and why pruning FFN neurons can selectively delete specific facts from a model.

Key Insight

The Transformer's two main components serve complementary roles. Attention moves information between token positions — it is a communication mechanism. The FFN processes information at each position independently — it is a computation and memory mechanism. Attention is "where to look"; FFN is "what to do with what you found."


import torch
import torch.nn as nn
import torch.nn.functional as F

class SwiGLU_FFN(nn.Module):
    """Feed-forward network with SwiGLU activation (LLaMA-style)."""
    def __init__(self, d_model: int, d_ff: int):
        super().__init__()
        self.w1 = nn.Linear(d_model, d_ff, bias=False)  # gate projection
        self.w3 = nn.Linear(d_model, d_ff, bias=False)  # up projection
        self.w2 = nn.Linear(d_ff, d_model, bias=False)   # down projection

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # SwiGLU: (Swish(W1 x) ⊙ W3 x) W2
        return self.w2(F.silu(self.w1(x)) * self.w3(x))

# Example: LLaMA-7B uses d_model=4096, d_ff=11008 (≈ 8/3 × 4096)
ffn = SwiGLU_FFN(d_model=4096, d_ff=11008)
x = torch.randn(1, 128, 4096)  # (batch=1, seq_len=128, d_model=4096)
out = ffn(x)  # (1, 128, 4096) — same shape in, same shape out
